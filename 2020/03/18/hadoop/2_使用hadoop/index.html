    1. 单机模式

    默认情况下，hadoop的配置是运行在非分布式的模式（单机模式），可以用一个例程来试一试：

    mkdir input
cp etc/hadoop/*.xml input
bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.2.jar grep input output 'dfs[a-z.]+'
cat output/*
    • 也就是使用hadoop这个应用程序
    • 执行一个jar包
    • 被执行的jar包的名字叫做share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.2.jar
    • 此时的主类叫做grep，同时这里主类也可以是wordcount
    • 后面的input和output分别是输入输出文件夹，其中输出文件夹必须是一个不存在的目录。
    • 'dfs[a-z.]+'这个是正则表达式，查询dfs开头的，后面跟1个或1个以上的字母，比如 dfsa dfsb dfsc

    该模式一般只用于调试测试，在实际的生产环境中不会使用本模式。

    下面测试一下wordcount

    bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.2.jar wordcount wcinput wcoutput

# 然后使用cat查看结果
cat wcoutput/*

    2. 伪分布式模式

    2.1 启动HDFS并运行MapReduce程序

    只有一个节点的分布式，叫做伪分布式，将所有节点运行在同一个机器上面。

    1. 配置集群

      1. etc/hadoop/core-site.xml:指定HDFS中namenode的地址

        注意，这里默认是没有配置的，默认表示使用本地模式，如果这里配置了，就使用指定的地址。

        下面的那个配置指定临时数据存储的位置

      <!-- 指定HDFS中NameNode的地址 -->
<property>
    <name>fs.defaultFS</name>
    <value>hdfs://hadoop001:9000</value>
</property>

<!-- 指定Hadoop运行时产生文件的存储目录 -->
<property>
    <name>hadoop.tmp.dir</name>
    <value>/opt/module/hadoop-2.7.2/data/tmp</value>
</property>
      1. etc/hadoop/hdfs-site.xml: 指定HDFS中副本的数量为1，因为只在一个节点上运行，虽然Hadoop是多副本机制，但是单一节点至多存储一个副本。
      <!-- 指定HDFS副本的数量 -->
<property>
    <name>dfs.replication</name>
    <value>1</value>
</property>

    2. 检查是否可以在没有密码的情况下SSH到本地主机

      ssh localhost

    3. 格式化namenode

      这个格式化只能在首次启动之前格式化

      bin/hdfs namenode -format

      要注意，不能多次格式化namenode。因为格式化NameNode会产生新的集群id,导致NameNode和DataNode的集群id不一致，集群找不到以往数据。所以，格式NameNode时，一定要先删除data数据和log日志，然后再格式化NameNode。

      sudo rm -rf logs
sudo rm -rf data

    4. 启动namenode和datanode

      sbin/start-dfs.sh

# 也可以通过下面两句分别启动
sbin/hadoop-daemon.sh start namenode
sbin/hadoop-daemon.sh start datanode

      此时使用jps就可以看到namenode和datanode都已经启动了

    查看HDFS文件系统

    首先需要在windows的host里面改一下映射

    192.168.186.130 hadoop001
192.168.186.131 hadoop002
192.168.186.132 hadoop003

    然后在浏览器里面查看

    hadoop001:50070

    操作集群

    1. 在HDFS文件系统上创建一个input文件夹

      bin/hdfs dfs -mkdir -p /user/hadoopUser/input

    2. 将测试文件内容上传到文件系统上

      bin/hdfs dfs -put wcinput/wc.input /user/hadoopUser/input/

    3. 查看上传的文件是否正确

      bin/hdfs dfs -ls /user/hadoopUser/input/
bin/hdfs dfs -cat /user/hadoopUser/input/wc.input

    4. 运行MapReduce程序

      bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.2.jar wordcount /user/hadoopUser/input/ /user/hadoopUser/output

    5. 查看输出结果

      bin/hdfs dfs -cat /user/atguigu/output/*

    2.2 启动YARN并运行MapReduce

    1. 配置集群

      1. 配置yarn：etc/hadoop/yarn-env.sh

        在下面的位置将 JAVA_HOME 给换掉

        # some Java parameters
export JAVA_HOME=/opt/module/jdk1.8.0_144

      2. 配置 yarn-site.xml

        <!-- Reducer获取数据的方式 -->
<property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
</property>

<!-- 指定YARN的ResourceManager的地址 -->
<property>
    <name>yarn.resourcemanager.hostname</name>
    <value>hadoop001</value>
</property>

      3. 配置 mapred-env.sh

        配置 JAVA_HOME

        export JAVA_HOME=/opt/module/jdk1.8.0_144

      4. 配置 mapred-site.xml (将 mapred-site.xml.template 重新命名为该文件) 

        mv mapred-site.xml.template mapred-site.xml

        然后添加配置：

        <!-- 指定MR运行在YARN上 -->
<property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
</property>

    2. 启动集群

      1. 启动前必须保证NameNode和DataNode已经启动

      2. 启动ResourceManager

        sbin/yarn-daemon.sh start resourcemanager

      3. 启动NodeManager

        sbin/yarn-daemon.sh start nodemanager

    3. 集群操作

      • 使用浏览器访问下面地址可以查看集群
      hadoop001:8088/cluster

      • 删除文件系统上的output文件

        bin/hdfs dfs -rm -R /user/hadoopUser/output

      • 执行MapReduce程序

        bin/hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.2.jar wordcount /user/hadoopUser/input  /user/hadoopUser/output

      • 查看运行结果

        bin/hdfs dfs -cat /user/hadoopUser/output/*

    2.3 配置历史服务器

    为了查看程序的历史运行情况，需要配置一下历史服务器。

    1. 配置 etc/hadoop/mapred-site.xml

      <!-- 历史服务器端地址 -->
<property>
    <name>mapreduce.jobhistory.address</name>
    <value>hadoop101:10020</value>
</property>
<!-- 历史服务器web端地址 -->
<property>
    <name>mapreduce.jobhistory.webapp.address</name>
    <value>hadoop101:19888</value>
</property>

    2. 启动历史服务器

      sbin/mr-jobhistory-daemon.sh start historyserver

    3. 查看历史服务器是否启动

      jps

    4. 查看JobHistory

      http://hadoop001:19888/jobhistory

    2.4 配置日志的聚集

    日志聚集概念：应用运行完成以后，将程序运行日志信息上传到HDFS系统上。

    日志聚集功能好处：可以方便的查看到程序运行详情，方便开发调试。

    注意：开启日志聚集功能，需要重新启动NodeManager 、ResourceManager和HistoryManager。

    1. 配置yarn-site.xml

      <!-- 日志聚集功能使能 -->
<property>
    <name>yarn.log-aggregation-enable</name>
    <value>true</value>
</property>

<!-- 日志保留时间设置7天 -->
<property>
    <name>yarn.log-aggregation.retain-seconds</name>
    <value>604800</value>
</property>

    2. 关闭NodeManager 、ResourceManager和HistoryManager

      sbin/yarn-daemon.sh stop resourcemanager
sbin/yarn-daemon.sh stop nodemanager
sbin/mr-jobhistory-daemon.sh stop historyserver

    3. 启动NodeManager 、ResourceManager和HistoryManager

      sbin/yarn-daemon.sh start resourcemanager
sbin/yarn-daemon.sh start nodemanager
sbin/mr-jobhistory-daemon.sh start historyserver

    4. 删除HDFS上已经存在的输出文件

      bin/hdfs dfs -rm -R /user/hadoopUser/output

    5. 执行WordCount程序

      hadoop jar share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.2.jar wordcount /user/hadoopUser/input /user/hadoopUser/output

    6. 查看日志

      http://hadoop001:19888/jobhistory

    2.5 配置文件说明

    Hadoop配置文件分两类：默认配置文件和自定义配置文件，只有用户想修改某一默认配置值时，才需要修改自定义配置文件，更改相应属性值。

    1. 默认配置文件

      要获取的默认文件 文件存放在Hadoop的jar包中的位置
      [core-default.xml] hadoop-common-2.7.2.jar/ core-default.xml
      [hdfs-default.xml] hadoop-hdfs-2.7.2.jar/ hdfs-default.xml
      [yarn-default.xml] hadoop-yarn-common-2.7.2.jar/ yarn-default.xml
      [mapred-default.xml] hadoop-mapreduce-client-core-2.7.2.jar/ mapred-default.xml

    2. 自定义配置文件

      • core-site.xml
      • hdfs-site.xml
      • yarn-site.xml
      • mapred-site.xml

      四个配置文件存放在$HADOOP_HOME/etc/hadoop这个路径上，用户可以根据项目需求重新进行修改配置。

    3. 全分布式模式

    3.1 scp实现集群安全拷贝

    scp可以实现服务器与服务器之间的数据拷贝。

    命令：

    scp    -r        $pdir/$fname            $user@hadoop$host:$pdir/$fname
命令   递归       要拷贝的文件路径/名称    目的用户@主机:目的路径/名称

    1. 在hadoop001上，将hadoop001中/opt/module目录下的软件拷贝到hadoop002上。

      scp -r /opt/module  root@hadoop002:/opt

    2. 在hadoop003上，将hadoop001服务器上的/opt/module目录下的软件拷贝到hadoop003上。

      sudo scp -r hadoopUser@hadoop001:/opt/module root@hadoop003:/opt

    3. 在hadoop004上，将hadoop001服务器上的/opt/module目录下的软件拷贝到hadoop003上。

      sudo scp -r hadoopUser@hadoop001:/opt/module root@hadoop004:/opt

    4. 将hadoop001中/etc/profile文件拷贝到hadoop002的/etc/profile上。

      sudo scp /etc/profile root@hadoop002:/etc/profile

    5. 将hadoop001中/etc/profile文件拷贝到hadoop003的/etc/profile上。

      sudo scp /etc/profile root@hadoop003:/etc/profile

    6. 将hadoop001中/etc/profile文件拷贝到hadoop004的/etc/profile上。

      sudo scp /etc/profile root@hadoop004:/etc/profile

    移动完成之后，要source一下/etc/profile

    3.2 rsync 远程同步工具

    rsync主要用于备份和镜像。具有速度快、避免复制相同内容和支持符号链接的优点。

    rsync和scp区别：用rsync做文件的复制要比scp的速度快，rsync只对差异文件做更新。scp是把所有文件都复制过去。

    命令：

    rsync    -rvl       $pdir/$fname              $user@hadoop$host:$pdir/$fname
命令     选项参数    要拷贝的文件路径/名称       目的用户@主机:目的路径/名称
    选项 功能
    -r 递归
    -v 显示复制过程
    -l 拷贝符号连接

    1. 把hadoop001机器上的/opt/software目录同步到hadoop002服务器的root用户下的/opt/目录

      rsync -rvl /opt/software/ root@hadoop002:/opt/software

    3.3 xsync集群分发脚本

    需求：循环复制文件到所有节点的相同目录

    1. rsync命令原始拷贝：

      rsync -rvl   /opt/module        root@hadoop003:/opt/

    2. 期望脚本：

      xsync 要同步的文件名称

      说明：在/home/hadoopUser/bin这个目录下存放的脚本，hadoopUser用户可以在系统任何地方直接执行。

      mkdir bin
cd bin/
touch xsync
vim xsync

    3. 在xsync中添加bash脚本：

      #!/bin/bash
#1 获取输入参数个数，如果没有参数，直接退出
pcount=$#
if((pcount==0)); then
    echo no args;
    exit;
fi

#2 获取文件名称
p1=$1
fname=`basename $p1`
echo fname=$fname

#3 获取上级目录到绝对路径
pdir=`cd -P $(dirname $p1); pwd`
echo pdir=$pdir

#4 获取当前用户名称
user=`whoami`

#5 循环
for((host=1; host<5; host++)); do
    echo ------------------- hadoop$host --------------
    rsync -rvl $pdir/$fname $user@hadoop00$host:$pdir
done

    4. 修改脚本 xsync 具有执行权限

      chmod 777 xsync

    5. 调用脚本形式：xsync 文件名称

      xsync /home/hadoopUser/bin

    在hadoop002上面进行如上操作，可以测试一下

    3.4 集群配置

    集群配置计划：

    hadoop002 hadoop003 hadoop004
    HDFS NameNode 、 DataNode DataNode SecondaryNameNode、DataNode
    YARN NodeManager ResourceManager NodeManager NodeManager

    1. 核心配置文件

      sudo vim etc/hadoop/core-site.xml

      编写以下指定配置：

      <!-- 指定HDFS中NameNode的地址 -->
<property>
    <name>fs.defaultFS</name>
    <value>hdfs://hadoop002:9000</value>
</property>

<!-- 指定Hadoop运行时产生文件的存储目录 -->
<property>
    <name>hadoop.tmp.dir</name>
    <value>/opt/module/hadoop-2.7.2/data/tmp</value>
</property>

    2. HDFS配置文件

      设置java版本

      sudo vim etc/hadoop/hadoop-env.sh
# 修改JAVA_HOME为如下内容
export JAVA_HOME=/opt/module/jdk1.8.0_144

      配置hdfs-site.xml

      sudo vim etc/hadoop/hdfs-site.xml

      然后在配置文件中添加以下内容

      <property>
    <name>dfs.replication</name>
    <value>3</value>
</property>

<!-- 指定Hadoop辅助名称节点主机配置 -->
<property>
    <name>dfs.namenode.secondary.http-address</name>
    <value>hadoop004:50090</value>
</property>

    3. YARN配置文件

      配置yarn-env.sh

      sudo vim etc/hadoop/yarn-env.sh
export JAVA_HOME=/opt/module/jdk1.8.0_144

      配置yarn-site.xm

      sudo vim etc/hadoop/yarn-site.xml

      然后在配置文件中添加以下内容

      <!-- Reducer获取数据的方式 -->
<property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
</property>

<!-- 指定YARN的ResourceManager的地址 -->
<property>
        <name>yarn.resourcemanager.hostname</name>
        <value>hadoop003</value>
</property>

    4. MapReduce配置文件

      配置mapred-env.sh

      sudo vim etc/hadoop/mapred-env.sh
export JAVA_HOME=/opt/module/jdk1.8.0_144

      配置mapred-site.xml

      # 从临时文件中拷贝出来修改后使用
cp mapred-site.xml.template mapred-site.xml
sudo vim etc/hadoop/mapred-site.xml

      配置mapred-site.xml

      <!-- 指定MR运行在Yarn上 -->
<property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
</property>

    5. 分发配置文件

      xsync /opt/module/hadoop-2.7.2/

    6. 检查配置文件是否分发成功

      cat /opt/module/hadoop-2.7.2/etc/hadoop/core-site.xml

      查看文件是否修改成功了。

      注意这里分发失败的原因可能是没有权限修改文件，需要通过chmod来加权限。

    3.5 集群单点启动

    1. 如果集群是第一次启动，需要格式化NameNode

      [hadoopUser@hadoop002 hadoop-2.7.2]$ hadoop namenode -format

    2. 启动namenode

      [hadoopUser@hadoop002 hadoop-2.7.2]$ hadoop-daemon.sh start namenode

    3. 在hadoop002、003和004上面都启动datanode

      [hadoopUser@hadoop002 hadoop-2.7.2]$ hadoop-daemon.sh start datanode

    3.8 停止集群

    以下指令如果不存在，说明环境没配好，需要添加sbin，或者直接在指令前面加上sbin也可以。

    1. 单点停止

      • HDFS

        hadoop-daemon.sh  start / stop  namenode / datanode / secondarynamenode

      • YARN

        yarn-daemon.sh  start / stop  resourcemanager / nodemanager

    2. 群起群停

      • HDFS

        start-dfs.sh   /  stop-dfs.sh

      • YARN

        start-yarn.sh  /  stop-yarn.sh
