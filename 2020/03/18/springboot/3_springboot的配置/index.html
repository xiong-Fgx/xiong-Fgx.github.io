    配置文件

    SpringBoot使用一个全局的配置文件：

    • application.properties
    • application.yml

    配置文件的名字是固定的，修改自动配置的默认值。放在 src/main/resources 目录下或者 类路径/config 下

    YAML配置

    1. YAML语法

    1. 使用缩进表示层级关系
    2. 缩进时不允许使用tab，只能用空格
    3. 缩进的空格数目不重要，只要相同层级有相同的缩进
    4. 大小写敏感

    数据结构：

    1. 对象：键值对的集合

      k: v，还是键值对的形式，在下一行来写对象的属性和值。

      friends:
    lastName: zhangsan
    age: 20

      也可以使用行内写法：

      friends: {lastName: zhangsan,age: 20}

    2. 数组：一组按次序排列的值

      短横线-表示数组中的一个元素

      pets:
    - cat
    - dog
    - pig

      行内写法：

      pets: [cat,dog,pig]

    3. 字面量：单个的、不可再分的值（数字、字符串、布尔类型）

      字符串默认不用加上单引号或者双引号。

      • 如果加了双引号，表示不会转义字符串里面的特殊字符，特殊字符会作为本身想表示的意思

        name: "hehe \n haha" 输出为：hehe换行haha

      • 如果是单引号，会转义特殊字符，特殊字符就变成普通的字符串

        name: 'hehe \n haha' 输出为：hehe \n haha

    2. SpringBoot导入yaml配置文件

    2.1 导入配置处理文件的依赖

    添加后，配置文件进行绑定时就会有提示

    <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-configuration-processor</artifactId>
    <optional>true</optional>
</dependency>

    2.2 新建Person类和Dog类

    注意，其中的 @ConfigurationProperties 注解的作用是：告诉SpringBoot将本类中的所有属性和配置文件的相关配置进行绑定。

    @Component
@ConfigurationProperties(prefix = "person")
public class Person {
    private String lastName;
    private Integer age;
    private Boolean boss;
    private Date birth;

    private Map<String, Object> maps;
    private List<Object> lists;
    private Dog dog;
    // toString和getter setter方法

public class Dog {
    private String name;
    private Integer age;
    // toString和getter setter方法

    2.3 编写application.yml

    在配置文件中将Person的各个属性进行指定。

    person:
  lastName: zhangsan
  age: 18
  boss: false
  birth: 2020/1/1
  maps: {k1: v1, k2: 12}
  lists:
    - lisi
    - zhaowu
  dog:
    name: hehe
    age: 2

    2.4 测试

    可以在测试期间实现很方便的类似容器注入的功能。

    在SpringBoot的单元测试类中，进行如下配置：

    @RunWith(SpringRunner.class)
@SpringBootTest
class Springboot001ApplicationTests {

    @Autowired
    Person person;

    @Test
    void contextLoads() {
        System.out.println(person);
    }
}

    就可以看到控制台输出了正确的值

    Person{lastName='zhangsan', age=18, boss=false, birth=Wed Jan 01 00:00:00 CST 2020, maps={k1=v1, k2=12}, lists=[lisi, zhaowu], dog=Dog{name='hehe', age=2}}

    Properties配置

    1. 写配置

    还是使用之前的配置，只不过将yaml文件换成properties文件进行配置

    可以看到直接就有补全了

    person.last-name=张三
person.age=18
person.birth=2020/1/1
person.boss=false
person.dog.name=hehe
person.dog.age=3
person.maps.k1=v1
person.maps.k2=15
person.lists=a,b,c

    需要注意的是，last-name加-表示大写，也可以使用latName，这一个特性叫做松散语法绑定。

    但是在 @Value 注解中只能使用 last-name，也就是Value注解不支持松散语法绑定。

    2. 乱码问题

    但是会发现输出的结果中有乱码：

    这是由于idea默认对于properties文件使用系统自带的编码，因此需要修改配置

    再运行就没有乱码了。

    3. ConfigurationProperties与Value注解

    除了使用 @ConfigurationProperties 注解进行注入之外，还可以使用 @Value 注解

    @Component
public class Person {
    @Value("${person.last-name}")
    private String lastName;
    @Value("#{11*2}")
    private Integer age;
    @Value("true")
    private Boolean boss;
    private Date birth;

    private Map<String, Object> maps;
    private List<Object> lists;
    private Dog dog;

    可以看出，如果使用 @ConfigurationProperties 注解，就会将所有的内容注入，但是使用 @Value 注解虽然需要一个一个指定，但是可选择某些不指定。

    4. 三个常用注解

    4.1 @PropertySource注解

    之前写配置文件的时候都是放在application.properties文件下，这个文件是SpringBoot的配置文件，但是有些配置是与SpringBoot无关的，不希望将这些配置放进该文件，这时就需要使用这一配置来指定配置的其他路径。

    例如，将person的信息全部放到person.properties文件中，此时对于Person类来说，就需要添加这条注解。

    @Component
// 添加这个注解，用于指定properties文件位置，可以是一个数组
@PropertySource(value = {"classpath:person.properties"})
@ConfigurationProperties(prefix = "person")
public class Person {
    private String lastName;
    private Integer age;
    private Boolean boss;
    private Date birth;

    private Map<String, Object> maps;
    private List<Object> lists;
    private Dog dog;

    4.2 @ImportResource注解

    导入Spring的配置文件，让配置文件里面的内容生效。

    4.3 @Bean注解

    如果采用全注解的方式，之前使用 @Configuration 注解，指明当前类是一个配置类，替代之前的Spring配置文件。在配置文件中，使用bean标签来添加组件。

    它作用是将方法的返回值添加到容器中，容器中这个组件的默认id就是方法名。

    配置文件占位符

    配置文件占位符在YAML和properties配置文件中均可使用，使用方式如下：

    可使用random来获取随机数，也可以通过引用前面的配置内容来获取配置文件中前面的配置。

    person.last-name=李四
person.age=${random.int}
person.dog.name=${person.last-name}_hehe
person.dog.age=${person.age:20}_hehe
    • ${}：其中的内容就是一个表达式，可以是函数，也可以是前文出现的配置
    • 如果没有属性，则会表现为字面意义
    • 可以设置默认值，就是在属性后面加冒号，后面跟默认值

    Profile

    Profile是SpringBoot对不同环境提供不同配置功能的支持，可以通过激活、指定参数等方式快速切换环境。

    1. 多Profile文件

    在主配置文件编写的时候，文件名可以是application-{profile}.properties/yml

    例如，在开发环境下，使用application-dev.properties，在生产环境下，使用application-prod.properties，在里面可以配置不同的端口之类的属性。默认使用的是application.properties

    激活方式

    在application.properties配置文件中指定要激活哪种方式

    spring.profiles.active=dev

    2. yml支持多文档块方式

    使用三个横线—可以将文档分隔，并且在每个文档块中定义其环境，使用profiles

    server:
  port: 8081
spring:
  profiles:
    active: dev
---
server:
  port: 8082
spring:
  profiles: dev
---
server:
  port: 8083
spring:
  profiles: prod

    激活方式

    在第一个文档块中添加active，表示激活的项。

    spring:
  profiles:
    active: dev

    3. 其他激活方式

    1. 可以使用命令行的方式指定：--spring.profiles.active=dev（打成jar包后运行时指定）
    2. 使用虚拟机参数：-Dspring.profiles.active=dev

    配置文件的加载位置

    SpringBoot启动会扫描以下位置的application.properties/yml文件作为SpringBoot的默认配置

    • file:./config/

    • file:./

      （上面两个file:位置就是和src同级的目录）

    • classpath:/config/

    • classpath:/

    以上是按照优先级从高到低的顺序，所有位置的文件都会被加载，采取互补的配置形式。

    高优先级配置内容会覆盖低优先级配置内容。

    也可以通过spring.config.location来改变默认配置。
