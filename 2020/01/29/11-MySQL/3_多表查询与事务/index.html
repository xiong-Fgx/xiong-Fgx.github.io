    多表查询

    查询语法：

    SELECT
    列名
FROM
    表名
WHERE
    ...

    首先，在这里使用两个表：dept和emp

    CREATE TABLE dept (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(20)
);

INSERT INTO dept (name) VALUES ('市场部'), ('开发部'), ('财务部');

CREATE TABLE emp (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(20),
    gender CHAR(1),
    salary DOUBLE,
    join_date DATE,
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES dept(id)
);
INSERT INTO emp (name, gender, salary, join_date, dept_id) VALUES ('孙悟空', '男', 7200, '2013-02-14', 1);
INSERT INTO emp (name, gender, salary, join_date, dept_id) VALUES ('猪八戒', '男', 3500, '2013-02-14', 2);
INSERT INTO emp (name, gender, salary, join_date, dept_id) VALUES ('唐僧', '男', 9000, '2013-02-14', 2);
INSERT INTO emp (name, gender, salary, join_date, dept_id) VALUES ('白骨精', '女', 4200, '2013-02-14', 3);
INSERT INTO emp (name, gender, salary, join_date, dept_id) VALUES ('蜘蛛精', '女', 3700, '2013-02-14', 1);

    经过初始化之后，可以得到下面的两张表：

    1. emp表

    2. dept表

    1. 普通的多表查询

    直接在查询语句中，在FROM后面写上多个表即可：

    SELECT * FROM emp, dept;

    查询到的结果是这样的：

    也就是说，查询结果的项数量是原来两个表中项目数量的乘积，被称为笛卡尔积。因为笛卡尔积中有很多无用的数据，因此要完成多表查询，需要消除无用的数据

    2. 内连接查询

    2.1 隐式内连接

    隐式内连接就是使用WHERE条件来消除无用的数据。

    例如上面那个表格中，emp表中的dept_id实际上是与dept表中的id表示相同的含义，因此可以通过这个来消除重复

    SELECT * FROM emp, dept WHERE emp.dept_id = dept.id;

    这样，查询到的结果都是emp表的dept_id属性与dept表的id属性相同的记录了。

    如果想要查询指定表中的指定数据，则只需要通过 . 运算符来指定表和列就可以了。

    SELECT emp.name, emp.gender, dept.name FROM emp, dept WHERE emp.dept_id = dept.id;

    注意事项：（需要确定的事情）

    1. 从哪些表中查询数据
    2. 查询条件是什么
    3. 查询字段是什么

    2.2 显式内连接

    语法：

    SELECT 字段列表 FROM 表名1 [INNER] JOIN 表名2 ON 条件;

    例如下面的：

    SELECT * FROM emp INNER JOIN dept ON emp.dept_id = dept.id;
SELECT * FROM emp JOIN dept ON emp.dept_id = dept.id;

    INNER是一个可选的操作，如果不写也是可以的

    3. 外连接查询

    分为两种：

    1. 左外连接
    2. 右外连接

    3.1 左外连接

    查询的是左表中的所有数据以及其交集部分，而内连接查询的是交集部分。

    语法：

    SELECT 字段列表 FROM 表1 LEFT [OUTER] JOIN 表2 ON 条件;

    OUTER是一个可选的操作，不写也是可以的。它与显式内连接的写法非常相似，只有几个关键字不同。

    3.2 右外连接

    查询的时右表中的所有数据以及交集部分，使用方法与左外连接是一样的。

    语法：

    SELECT 字段列表 FROM 表1 RIGHT [OUTER] JOIN 表2 ON 条件;

    因此：左外与右外查询实际上是一种，只不过表的位置写的不一样而已。

    4. 子查询

    查询中嵌套查询，就称嵌套的查询叫做子查询，根据子查询的结果情况，可以分为三种，如下所示。

    4.1 子查询的结果是单行单列

    子查询的结果可以作为条件，使用运算符。

    例如：要查询工资最多的员工信息，如果不使用子查询，则需要分为两步：先确定最大工资，然后通过最大工资来查询，即：

    SELECT MAX(salary) FROM emp;
-- 得到结果为9000
SELECT * FROM emp WHERE emp.salary = 9000;
-- 使用上一步得到的结果9000进行查询

    如果使用子查询，就可以将上面的两步查询合二为一

    SELECT * FROM emp WHERE emp.salary = (SELECT MAX(salary) FROM emp);

    4.2 子查询的结果是多行单列

    可以通过使用IN运算符来判断。

    例如：要查询部门为财务部和市场部两个部门的所有成员信息，则如果使用两条语句可以写成下面这样：

    SELECT id FROM dept WHERE name = '财务部' OR name = '市场部';
-- 得到的结果为id=2和id=3
SELECT * FROM emp WHERE dept_id = 2 OR dept_id = 3;
-- 使用上一步得到的结果2和3进行查询

    如果使用子查询，则可以将上面的两条查询语句合二为一，其中需要使用IN来从这个多行单列的子查询结果中获取查询条件。

    SELECT * FROM emp WHERE dept_id IN (SELECT id FROM dept WHERE name = '财务部' OR name = '市场部');

    4.3 子查询的结果是多行多列

    如果子查询的结果是多行多列，则可以将这个子查询的结果作为一个虚拟表，来进行表的查询。

    查询员工入职日期是2011年11月11日之后的员工信息和部门信息

    SELECT * FROM emp WHERE emp.join_date > '2011-11-11';
-- 把上面的子查询结果当成一张表
SELECT * FROM dept t1, (SELECT * FROM emp WHERE emp.join_date > '2011-11-11') t2
WHERE t1.id = t2.dept_id;
-- 这里就将前面子查询语句拿过来当做一张表，并将其起名为t2，就能够进行子查询结果为多行多列的查询了

    上面的查询也可以使用普通的内连接来查询：

    SELECT * FROM tmp t1, dept t2 WHERE t1.dept_id = t2.id AND t1.join_date > '2011-11-11';

    事务

    1. 事务的基本介绍

    概念：如果一个包含多个步骤的业务操作被事务管理，那么这些操作要么同时成功，要么同时失败。

    例如：张三给李四转账500元

    1. 查询张三账户的余额是否大于500
    2. 张三账户金额-500
    3. 李四账户金额+500

    如果在某第二步结束之后出现异常，则后面的操作将不会再进行，此时的影响是：张三的账户-500，但是李四的账户没有+500。

    如果这三个步骤被事务管理，首先会开启事务。一旦其中一个步骤出现异常，就会回滚，如果没有出现异常，则会提交事务。这三种操作在SQL中会有三条语句：

    • 开启事务：START TRANSACTION
    • 回滚：ROLLBACK
    • 提交事务：COMMIT

    使用一个实际的例子为：

    START TRANSACTION
UPDATE account SET balance = balance - 500 WHERE name = '张三';
UPDATE account SET balance = balance + 500 WHERE name = '李四';
-- 如果没有问题，则提交事务
COMMIT
-- 如果出现问题，则回滚
ROLLBACK

    事务提交的方式：

    • 自动提交

      mysql是自动提交的，一条DML语句就会自动提交一次事务

    • 手动提交

      需要手动开启事务，再提交

    修改事务的默认提交方式：

    • 查询默认的提交方式

      SELECT @@autocommit; -- 其中，1代表自动提交，0代表手动提交

    • 改变默认提交方式

      SET @@autocommit = 0;

    2. 事务的四点特征

    事务的四大特征

    1. 原子性：是不可分割的最小操作单位，要么同时成功，要么同时失败
    2. 持久性：事务提交或回滚之后，数据库会持久化保存数据
    3. 隔离性：多个事务之间相互独立
    4. 一致性：事务操作前后数据总量不变

    3. 事务的隔离级别

    概念：多个事务之间是隔离的，也就是说他们是独立的，但是多个事务操作同一批事务，则会产生一些问题，设置不同的隔离级别，就可以解决这些问题。

    3.1 问题

    1. 脏读

      一个事务读取到另一个事务中没有提交的数据

    2. 不可重复读（虚读）

      在同一个事务中，两次读取到的数据不一样

    3. 幻读

      一个事务操作（DML）数据表中的所有记录，另一个事务添加了一条数据，则第一个事务查询不到自己的修改

    3.2 隔离级别

    1. READ UNCOMMITTED

      即读未提交

      产生的问题：脏读、不可重复读、幻读

    2. READ COMMITTED（ORACLE的默认级别）

      读已提交，也就是提交之后其他事务才可以读取

      产生的问题：不可重复读、幻读

    3. REPEATABLE READ（MYSQL的默认级别）

      可重复读，设置了这个隔离级别之后，在事务未提交或回滚时，多次读取得到的结果是相同的

      产生的问题：幻读

    4. SERIALIZABLE

      即串行化，可以解决所有的问题，如果有两个事务来操作同一个表，则必须等待一个事务操作完毕之后，另一个事务才可以开始执行。也就相当于加了一个锁，使用的效率会大大降低，不过安全性很高。

    隔离级别从小到大，其安全性越来越高，但是其效率会越来越低

    3.3 查询与操作

    查询隔离级别：

    SELECT @@tx_isolation;

    设置隔离级别：

    SET GLOBAL TRANSACTION LEVEL 级别字符串;

    在设置完隔离级别之后，需要将连接断开然后重新连接，才会生效
