    1. 环境搭建步骤

    首先需要使用root用户登录，登录完成后执行以下几步：

    1.1 关防火墙

    service iptables stop
chkconfig iptables off

#检查防火墙是否被关闭：
service iptables status
chkconfig --list iptables

    1.2 创建一个一般用户，并更改密码

    useradd hadoopUser
passwd hadoopUser

    1.3 在opt目录下创建software、module文件夹，并改变所有权

    mkdir /opt/software /opt/module  
chown hadoopUser:hadoopUser /opt/software /opt/module 
# 注意这里的冒号前后是用户和组，写一样的就行了

    1.4 增加可以使用sudo命令的用户

    vim /etc/sudoers

    然后再下面这个位置增加和用户相关的信息：

    hadoopUser  ALL=(ALL)  NOPASSWD: ALL

    为了验证这个是否正确，可以将用户切换至hadoopUser然后执行一条sudo命令，如果没有报错，则表示该用户可以正常使用sudo命令

    1.5 更改hosts

    使用下面这条命令打开hosts文件并进行修改（注意，需要在root用户下进行修改，否则将不能保存修改）

    vim /etc/hosts

    修改之前需要先查询以下本机的IP地址，也就是使用ifconfig来看到

    在文件后面追加以下内容：

    注意修改的IP地址需要和自己的IP地址一样

    小提示：这里使用vim的指令：

    • yy表示复制一整行
    • dd表示删除一整行
    • p表示粘贴

    因此可以通过yy+p来快速进行写入

    1.6 改静态IP

    vim /etc/sysconfig/network-scripts/ifcfg-eth0

    然后将文件中的内容写如下所示：

    DEVICE="eth0"
BOOTPROTO="static"
IPADDR="192.168.57.134"
PREFIX="24"
GATEWAY="192.168.57.2"
ONBOOT="yes"
TYPE="Ethernet"
DNS1="192.168.57.2"
NAME="eth0"

    如果与下面有不一样的则全部都删掉，改成和这个一样的就可以了。其中网关设置，需要在VMware里面查看：编辑-虚拟网关编辑器，然后按照下面的方式查看，并且设置成网关IP，DNS地址设置和网关地址相同就可以了。

    1.7 改主机名

    vim /etc/sysconfig/network

    在这个文件中，将HOSTNAME换成指定的主机名就行了。

    注意：这里需要将hosts、network、ifconfig中看到的主机名与IP地址对应且相同！

    1.8 拍快照并克隆

    上面的操作完成之后，就已经完成了机器的配置，需要进行拍快照复制主机了。将当前的虚拟机关闭，并且拍摄一张快照configureOK，然后从configureOK处克隆快照。

    克隆完毕之后，打开这个克隆出来的新虚拟机，发现在登录界面就提示以下的错误：

    因为之前设置的ifcfg里面的网卡名称叫做eth0，只有一个网卡，并且被之前的虚拟机占用了。也就是说eth0这个网卡需要重新配置。并且，主机名、静态IP也需要重新指定。

    1.9 修改新虚拟机的配置

    使用下面的命令来对文件进行修改，主要就是：

    1. 删除网卡eth0（这是之前虚拟机的网卡）
    2. 修改静态IP地址
    3. 修改本机的主机名

    前面三个步骤依次对以下三个文件进行相关的修改，主要也就是把IP地址和主机名给换掉。

    sudo vim /etc/udev/rules.d/70-persistent-net.rules
sudo vim /etc/sysconfig/network-scripts/ifcfg-eth0
sudo vim /etc/sysconfig/network

    注意：要使用sudo，否则无法写该文件。

    改完之后重启，发现之前报的错已经没了

    5. 安装java与hadoop

    5.1 拷贝并解压tar包

    往里面传文件的时候直接将文件拖动到xshell里面就可以了，如果上传不过去，可以在虚拟机里面安装一个软件sudo yum install lrzsz。

    使用下面的命令将考进去的tar包解压

    tar -zxvf jdk-8u1... -C /opt/module/

    其中的参数含义为：

    • z：gzip的压缩编码
    • x：解压
    • v：解压过程中显示详细信息
    • f：指定解压文件
    • -C：指定解压目录

    将这两个tar包都放到module文件夹下

    5.2 配置环境变量

    配置环境变量：

    sudo vim /etc/profile

    使用sudo对profile文件进行编辑，在最后一行加上以下两句话。

    export JAVA_HOME=/opt/module/jdk1.8.0_144
export PATH=$PATH:$JAVA_HOME/bin

    然后用source执行该配置文件

    source /etc/profile

    此后，再进行echo $JAVA_HOME的时候，就能够打印出刚才配置出来的结果了。然后可以查看java的版本，应该是能够看到的

    对于hadoop也是一样的，只不过需要注意要加一个sbin。

    export HADOOP_HOME=/opt/module/hadoop-2.7.2
export PATH=$PATH:$HADOOP_HOME/bin
export PATH=$PATH:$HADOOP_HOME/sbin

    然后查询hadoop的版本就可以得到下面的输出

    下面看一下hadoop文件夹下的内容

    • bin：可执行文件
    • etc：配置文件
    • include：头文件
    • lib：库
    • libexec：
    • sbin：shellbin，可执行脚本
    • share：所有的jar包都放在该目录下
