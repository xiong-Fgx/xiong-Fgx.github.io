    HTTP

    1. server端

    import com.sun.net.httpserver.HttpExchange;
import com.sun.net.httpserver.HttpHandler;
import com.sun.net.httpserver.HttpServer;

    一个简单的Http Server端需要导入以上三个包，而且大概也只需要这三个就可以完成基本的操作。其中HttpExchange用来定义或控制一个Http交互中的请求与响应内容。

    1.1 HttpExchange

    文档中的介绍如下：

    此类封装了在一个交换中，可以用在一个交换中的请求与响应两个阶段，其作用如下：

    • 请求阶段：解析接收到的HTTP请求，检查来自客户端的请求
    • 响应阶段：要生成的响应，构建和发送响应

    主要的方法：

    • 请求阶段：

      1. getRequestMethod() ：确定命令

        HttpExchange exchange;
String method = exchange.getRequestMethod();

      2. getRequestHeaders() ：检查请求标头（如果需要）

      3. getRequestBody() ：返回一个 InputStream 用于读取请求正文。 读取请求正文后，流关闭。

        HttpExchange exchange;
String bodyStr = new String(exchange.getRequestBody().readAllBytes());
bodyStr = URLDecoder.decode(bodyStr, "UTF-8");

    • 响应阶段：

      1. getResponseHeaders() ：设置任何响应头，内容长度除外

      2. [sendResponseHeaders(int,long)](https://docs.oracle.com/javase/7/docs/jre/api/net/httpserver/spec/com/sun/net/httpserver/HttpExchange.html#sendResponseHeaders(int, long)) ：发送响应头。 必须在下一步之前调用。

        exchange.sendResponseHeaders(200, 0);

      3. getResponseBody() 获得一个 OutputStream ，发送响应正文。 写入响应主体后，必须关闭流以终止交换。

        exchange.getResponseBody().write("success".getBytes());
exchange.close();

    1.2 HttpHandler

    文档中的描述是：

    被调用以处理HTTP交换的处理程序。 每个HTTP交换均由这些处理程序之一处理。

    也就是说，每个HTTP Exchange 交互过程都需要在这个接口的实现中处理。

    在java中创建一个类，并且这个类用来实现HttpHandler接口，并且重写handler方法：

    class 类名 implements HttpHandler {

    @Override
    public void handle(HttpExchange exchange) throws IOException {

    }
}

    因此，下面是一个例子，其中重写了HttpHandler接口，实现出来了两个类，这两个类分别用来处理不同的Http请求。

    class FormHttpHandler implements HttpHandler {

    @Override
    public void handle(HttpExchange exchange) throws IOException {
        System.out.println("from方法：" + exchange.getRequestMethod());//get
        String paramStr = exchange.getRequestURI().getQuery();
        System.out.println("get参数：" + TestHttpServer.getParams(paramStr));
        exchange.sendResponseHeaders(200, 0);
        String body = "get Message!!!";
        exchange.getResponseBody().write(body.getBytes());
        exchange.close();
    }

}
class InfoHttpHandler implements HttpHandler {

    @Override
    public void handle(HttpExchange exchange) throws IOException {
        System.out.println("info方法：" + exchange.getRequestMethod());//post
        String bodyStr = new String(exchange.getRequestBody().readAllBytes());
        bodyStr = URLDecoder.decode(bodyStr, "UTF-8");
        System.out.println("post收到：" + TestHttpServer.getParams(bodyStr));
        exchange.sendResponseHeaders(200, 0);
        exchange.getResponseBody().write("success".getBytes());
        exchange.close();
    }
}

    1.3 HttpServer

    对于HttpServer的使用，需要进行以下两个步骤：

    • 建立一个HttpServer对象

      此类实现一个简单的HTTP服务器。 HttpServer绑定到IP地址和端口号，并侦听此地址上来自客户端的传入TCP连接。 子类HttpsServer实现了一个处理HTTPS请求的服务器。

      HttpServer server = HttpServer.create(new InetSocketAddress(8080), 0);

      上面cteate函数的第二个参数为socket backlogs，这个参数表示系统将在内部排队的最大传入TCP连接数。

      连接在等待HttpServer接受时排队。当达到限制时，基础的TCP实现可能会拒绝（或可能忽略）其他连接，因此需要合理地设置该值。

    • 关联HttpHandler对象

      服务器必须与一个或多个HttpHandler对象关联才能处理请求。每个此类HttpHandler都注册有一个根URI路径，该路径表示应用程序或服务在此服务器上的位置。 处理程序到HttpServer的映射由HttpContext对象封装。

      • HttpContext是通过调用 createContext(String，HttpHandler) 创建的。
      • 找不到任何处理程序的任何请求都会被404响应拒绝。

      通过提供Executor对象，可以在此对象外部进行线程管理。 如果未提供，则使用默认实现。例如下面这个例子，就是来对特定的URI需要进行的不同操作进行绑定。

      server.createContext("/form", new FormHttpHandler());
server.createContext("/info", new InfoHttpHandler());

      也就是说，经过上面的绑定操作之后，如果server收到了URI为 /form 的请求，将会使用 FormHttpHandler 类中重写的 handle 方法进行处理。

      该函数的使用原理如下：

      建立关联

      收到HTTP请求时，将请求URI映射到HttpContext路径。通过查找其路径是请求URI路径的最长匹配前缀的上下文，可以找到适当的HttpContext（和处理程序）。比较字符串时会区分大小写，并且不会与任何编码形式进行转换。 例如。 给定配置了以下HttpContext的HttpServer。

      context context type
      ctx1 “/“
      ctx2 “/apps/“
      ctx3 “/apps/foo/“

      然后，假如收到了下面的请求的URI，则与之匹配的context分别如下：

      请求的URI context
      http://foo.com/apps/foo/bar ctx3
      http://foo.com/apps/Foo/bar 无匹配，错误
      http://foo.com/apps/app1 ctx2
      http://foo.com/foo ctx1

    • 启动HttpServer

      这一步没啥好说的，就是调用start()方法来启动HttpServer

      server.start();

    public static Map<String, String> getParams(String paramStr){

        if(paramStr != null) {
            return Stream.of(paramStr.split("&"))
                    .filter(s -> s.split("=").length == 2 && s.split("=")[0].length() > 0 && s.split("=")[1].length() > 0)
                    .collect(Collectors.toMap(x -> x.toString().split("=")[0], y -> y.toString().split("=")[1]));

        return null;
    }

    这里主要就是进行一些必要的初始化，写在静态代码块中，一旦类被使用，这部分代码将被执行，其操作的各种变量均为静态变量，因为HTTP client初始化的内容对于所有类都是共享的，不需要将这些内容设置为类变量，同时用起来会非常方便。

    static {

    try {

        SSLContextBuilder builder = new SSLContextBuilder();
        builder.loadTrustMaterial(null, new TrustSelfSignedStrategy());
        SSLConnectionSocketFactory sslsf =

        // 配置同时支持 HTTP 和 HTPPS

    } catch (NoSuchAlgorithmException e) {

    2.4.0 包装POST请求

    首先，如果需要发送一个POST请求，需要new一个HttpPost的对象，然后将这个对象作为参数分别传递给2.4.1~2.4.5这些函数中进行实际的发送与交互处理。

    private static String sendHttpPost(HttpPost httpPost) {

    CloseableHttpClient httpClient = null;
    CloseableHttpResponse response = null;
    // 响应内容
    String responseContent = null;
    try {
        // 创建默认的httpClient实例.
        httpClient = getHttpClient();
        // 配置请求信息
        httpPost.setConfig(requestConfig);
        // 执行请求
        response = httpClient.execute(httpPost);
        // 得到响应实例
        HttpEntity entity = response.getEntity();

        // 可以获得响应头
        // Header[] headers = response.getHeaders(HttpHeaders.CONTENT_TYPE);
        // for (Header header : headers) {
        // System.out.println(header.getName());
        // }

        // 得到响应类型
        // System.out.println(ContentType.getOrDefault(response.getEntity()).getMimeType());

        // 判断响应状态
        if (response.getStatusLine().getStatusCode() >= 300) {
            throw new Exception(
                "HTTP Request is not success, Response code is " + 
                response.getStatusLine().getStatusCode());
        }

        if (HttpStatus.SC_OK == response.getStatusLine().getStatusCode()) {

        }

}

public static String sendHttpPost(String httpUrl, Map<String, String> maps) {
    String parem = convertStringParameter(maps);
    return sendHttpPost(httpUrl, parem);
}

    其中，进行参数转换的convertStringParameter(maps)函数的定义如下：

    /**
 * 将map集合的键值对转化成：key1=value1&key2=value2 的形式
 *
 * @param parameterMap 需要转化的键值对集合
 * @return 字符串
 */
public static String convertStringParameter(Map parameterMap) {
    StringBuffer parameterBuffer = new StringBuffer();
    if (parameterMap != null) {
        Iterator iterator = parameterMap.keySet().iterator();
        String key = null;
        String value = null;
        while (iterator.hasNext()) {
            key = (String) iterator.next();
            if (parameterMap.get(key) != null) {
                value = (String) parameterMap.get(key);
            } else {
                value = "";
            }
            parameterBuffer.append(key).append("=").append(value);
            if (iterator.hasNext()) {
                parameterBuffer.append("&");
            }
        }
    }
    return parameterBuffer.toString();
}

    2.4.3 带参数和文件

    public static String sendHttpPost(String httpUrl, 
                                  Map<String, String> maps, 
                                  List<File> fileLists) 
{
    HttpPost httpPost = new HttpPost(httpUrl);// 创建httpPost
    MultipartEntityBuilder meBuilder = MultipartEntityBuilder.create();
    if (maps != null) {
        for (String key : maps.keySet()) {
            meBuilder.addPart(key, 
                              new StringBody(
                                  maps.get(key), 
                                  ContentType.TEXT_PLAIN
                              ));
        }
    }
    if (fileLists != null) {
        for (File file : fileLists) {
            FileBody fileBody = new FileBody(file);
            meBuilder.addPart("files", fileBody);
        }
    }
    HttpEntity reqEntity = meBuilder.build();
    httpPost.setEntity(reqEntity);
    return sendHttpPost(httpPost);
}

        // System.out.println(header.getName());
        // }

        // 得到响应类型
        // System.out.println(ContentType.getOrDefault(response.getEntity()).getMimeType());

        // 判断响应状态
        if (response.getStatusLine().getStatusCode() >= 300) {
            throw new Exception(
                "HTTP Request is not success, Response code is " + 
                response.getStatusLine().getStatusCode());
        }

        if (HttpStatus.SC_OK == response.getStatusLine().getStatusCode()) {
            responseContent = EntityUtils.toString(entity, CHARSET_UTF_8);
            EntityUtils.consume(entity);
        }

    } catch (Exception e) {
        e.printStackTrace();
    } finally {
        try {
            // 释放资源
            if (response != null) {
                response.close();
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
    return responseContent;
}
