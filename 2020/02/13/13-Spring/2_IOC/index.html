    IoC

    将创建对象的控制权交给工厂，也就是可以通过工厂模式实现IoC，是框架的一个重要特性。

    IoC包括了依赖注入 DI(Dependency Injection) 和依赖查找 DL(Dependency Lookup)

    作用是削减计算机程序之间的耦合，解除代码中的依赖关系。

    本质：Spring的IoC实质上是一个Map类型的容器，其key是String类型，value是Object类型，通过String的索引来找Spring创建出来对应的对象。

    1. 配置

    在pom文件中配置spring，增加下面这个dependency：

    <dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-context</artifactId>
    <version>5.2.3.RELEASE</version>
</dependency>

    在resources中添加一个bean.xml，在其中配置IoC

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--id：获取时的唯一标识，class：反射时的全限定类名-->
    <bean id="accountDao" class="com.xiong.dao.Impl.AccountDaoImpl"></bean>
    <bean id="accountService" class="com.xiong.service.impl.AccountServiceImpl"></bean>

</beans>

    在模拟的clinet中做一些改变，需要获取spring的IoC核心容器，并且根据id获取对象。

    注意，通过核心容器，并且根据id获取对象的方法有两种：

    • 拿到Object类型并进行强制类型转换
    • 直接给getBean方法传入类型的字节码，直接获取转换后类型的对象
    public class client {
    public static void main(String[] args) {
        // 1. 获取核心容器对象
        ApplicationContext ac = new ClassPathXmlApplicationContext("bean.xml");
        // 2. 根据id获取bean对象
        // 方法1：拿到Object类型并进行强制类型转换
        AccountService as = (AccountService) ac.getBean("accountService");
        // 方法2：通过第二个参数：字节码，进行强制转换
        AccountDao ad = ac.getBean("accountDao", AccountDao.class);
        System.out.println(as);
        System.out.println(ad);
    }
}

    这样就把前面jdbc例子里面使用BeanFactory创建对象的功能交给Spring来做了。

    2. 核心容器的三个实现类

    • ClassPathXmlApplicationContext

      可以加载类路径下的配置文件，要求配置文件必须在类路径下，不在的话，就不能加载

      ApplicationContext ac = new ClassPathXmlApplicationContext("bean.xml");

    • FileSystemXmlApplicationContext

      可以加载磁盘任意路径下的配置文件（必须有访问权限）

      ApplicationContext ac = new FileSystemXmlApplicationContext("C:\\Users\\52240\\Desktop\\bean.xml");

    • AnnotationConfigApplicationContext

      读取注解创建容器

    3. BeanFactory与ApplicationContext

    这是两种创建核心容器的方法，下面详细介绍：

    • ApplicationContext

      创建核心容器时，创建对象采取的策略是立即加载的方式。也就是说，只要一读取完配置文件就马上创建配置文件中配置的对象。

      使用场景：单例对象

      ApplicationContext ac = new ClassPathXmlApplicationContext("bean.xml");
AccountService as = (AccountService) ac.getBean("accountService");
AccountDao ad = ac.getBean("accountDao", AccountDao.class);

    • BeanFactory

      创建核心容器时，创建对象采取的策略是延迟加载的方式。也就是说，什么时候根据id获取对象，什么时候才真正创建对象。

      使用场景：多例对象

      Resource resource = new ClassPathResource("bean.xml");
BeanFactory factory = new XmlBeanFactory(resource);
AccountService as = (AccountService) factory.getBean("accountService");

    4. 创建bean对象的方式

    创建bean有三种方式，在bean.xml中进行创建

    • 使用默认构造函数创建

      在配置文件中使用bean标签配置id和class属性之后，且没有其他属性和标签时，采用的就是默认构造函数创建bean对象。此时如果类中没有默认构造函数，则对象无法创建。

      <bean id="accountService" class="com.xiong.service.impl.AccountServiceImpl"></bean>

      此时调用的时默认构造函数

      相反，如果没有这个无参的构造函数，则会报错。

    • 使用普通工厂中的方法创建对象（使用某个类中的方法创建对象并存入spring容器）

      这种情况一般发生在有外部提供了jar包中带有普通工厂，而我们希望通过这个工厂获取对象，但是因为jar包中的类是不能够修改的，因此如果还想要使用spring容器获取，可以使用这种方式来创建。例如下面这个，在InstanceFactory中有一个getAccountService方法可以返回一个实例对象。

      <bean id="instanceFactory" 
      class="com.xiong.factory.InstanceFactory"></bean>
<bean id="accountService" 
      factory-bean="instanceFactory"
      factory-method="getAccountService"></bean>

      首先说明instanceFactory的类是com.xiong.factory.InstanceFactory，然后将它写在下面的factory-bean参数中。这样就可以通过一个普通的工厂函数获取对象了

    • 使用工厂中的静态方法创建对象（使用某个类的静态方法创建对象，并存入spring容器）

      此时的bean.xml文件应该写成这样：

      <bean id="accountService" 
      class="com.xiong.factory.StaticFactory" 
      factory-method="getAccountService"></bean>

    5. bean对象的作用范围

    通常情况下，bean是一个单例的bean，也就是通过核心容器的getBean方法获取同一个id得到的对象也是相同的，可以通过下面的方法进行测试：

    ApplicationContext ac = new ClassPathXmlApplicationContext("bean.xml");
AccountService as = (AccountService) ac.getBean("accountService");
AccountService as2 = (AccountService) ac.getBean("accountService");
System.out.println(as);
System.out.println(as2);

    在bean标签的scope属性中有多个取值，可以设置bean不同的作用范围：

    • singleton：单例，也是默认值
    • prototype：多例
    • request：作用于web应用的请求范围
    • session：作用于web应用的会话范围
    • global-session：作用于集群环境的会话范围（全局会话范围），当不是集群环境的时候，就是session

    6. bean对象的生命周期

    根据是单例对象或者多例对象，bean有不同的生命周期

    • 单例对象

      1. 出生：当核心容器创建的时候，对象就产生了
      2. 活着：只要容器还在，对象就一直活着
      3. 死亡：容器销毁，对象死亡

      也就是说：单例对象的生命周期和核心容器相同。

      可以通过下面这个例子，在xml文件中指定init和destroy

      <bean id="accountService"
      scope="singleton"
      init-method="init"
      destroy-method="destroy"
      class="com.xiong.service.impl.AccountServiceImpl"></bean>

      同时也需要在相应的类实现文件中写上对应的方法：

      但是运行的结果中只执行了init方法，并没有执行destroy方法，这是因为主函数的流程，还没有调用销毁方法，就已经将容器释放了，因此需要手动调用容器的关闭方法才能够执行destroy。

      注意，这里的ac如果还是ApplicationContext接口的话是不同调用close方法的，因为close方法属于ClassPath…这个类的，因此需要注意红圈中的部分，要把ac换成这一个类型。

    • 多例对象

      1. 出生：使用对象的时候spring框架为我们创建对象，针对上面那个例子，就是执行getBean的时候才可以
      2. 活着：只要在使用过程中就一直活着
      3. 死亡：由于spring框架无法得知何时对象不使用，因此不会自动将其销毁。所以当对象长时间不使用，且没有别的对象引用时，由Java的GC进行回收。

      总的来说：多例对象使用的时候才创建，并且只能被回收而消亡，不能手动令其死亡。

    依赖注入

    依赖注入：Dependency Injection

    IoC的作用是降低耦合，依赖关系的管理都交给了spring来维护。在当前类中需要用到其他类的对象，只需要在配置文件中说明，依赖关系的维护就叫做依赖注入。

    能注入的类型有三类：

    • 基本类型和String
    • 其他的bean类型（在配置文件中或者注解配置过的bean）
    • 复杂类型（集合类型）

    注入的方式

    • 使用构造函数提供
    • 使用set方法提供
    • 使用注解提供

    1. 构造函数注入

    因为通过xml配置文件中配置的话，会比较麻烦，因为数据可能会经常变化。因此需要通过构造函数进行注入。

    下面将构造函数写成下面这样：

    构造函数注入的几个关键点：

    • 使用的标签是constructor-arg，位于bean标签内部。

    • 标签中的属性：

      1. type：用于指定要注入的数据的类型，该属性类型也是构造函数中某个或某些参数的类型。例如：

        <constructor-arg type="java.lang.String" value="test"></constructor-arg>

        上面的这个标签就会将"test"这个字符串赋给String类型的成员。如果有两个String有两个，则不能实现独立注入的功能

      2. index：用于指定要注入的数据给构造函数中指定索引位置的参数赋值。（索引的位置是从0开始的）

      3. name：用于指定给构造函数中指定名称的构造函数赋值

        =============以上三个指定给构造函数中那个参数赋值，常用的就是name==============

      4. value：用于给基本类型和String类型

      5. ref：用于指定其他的Bean类型数据，它指的就是在spring的核心容器中出现过的Bean对象。

    对于上面那个例子，构造函数中有三个参数，就可以通过下面的方式来写

    <bean id="accountService" class="com.xiong.service.impl.AccountServiceImpl">
    <constructor-arg name="name" value="test"></constructor-arg>
    <constructor-arg name="age" value="18"></constructor-arg>
    <constructor-arg name="birthday" ref="now"></constructor-arg>
</bean>
<bean id="now" class="java.util.Date"></bean>

    注意，其中Date类型不属于基本类型或String，需要先配置将其存入核心容器中，然后被ref指定。

    最后一行的含义是：读取全限定类名java.util.Date，反射创建一个对象，并且存入spring的容器中。可以通过它创建出来

    构造函数注入的优点：

    • 获取bean对象时，注入数据是必须的操作，否则对象无法创建成功。

    缺点：

    • 改变了bean对象的实例化方式，使我们在创建对象时，必须将所有的参数全部注入，但是如果用不到这些数据，也必须提供，就造成了浪费。

    2. set方法注入

    下面重新创建一个类，还是用之前AccountServiceImpl类，但是把构造函数去掉，并添加相应的set方法（生成各个属性的set方法就可以了）

    xml文件中需要进行如下改动：

    • 涉及的标签：property

    • 出现的位置：bean标签内部

    • 标签中的属性：

      1. name：用于指定注入时所调用的set方法的名称

        注意与属性名称无关，至于set方法的名称相关，需要写成方法名称去掉前面的set，并且将大写换成小写。

        例如前面用于设置name的函数叫做setName，那么，先去掉set，然后把N换成n，也就是写成name，同理，如果设置name的函数叫做setUsername，那么就要写成username

      2. value：用于给基本类型和String类型

      3. ref：用于指定其他的Bean类型数据，它指的就是在spring的核心容器中出现过的Bean对象。

    下面是一个具体使用的例子：

    <bean id="accountService1" class="com.xiong.service.impl.AccountServiceImpl1">
    <property name="name" value="test"></property>
    <property name="age" value="21"></property>
    <property name="birthday" ref="now"></property>
</bean>
<bean id="now" class="java.util.Date"></bean>

    set方法注入的优势：

    • 创建对象的时候没有明确地限制，可以直接使用默认构造函数

    缺点：

    • 如果有某个成员必须有值，则set方法无法保证一定注入，也就是获得对象时可能set方法没有执行

    相比较于构造函数注入，更常用的时set方法注入。

    3. 集合数据注入

    新建一个类，该类中有各种集合数据，然后在xml中配置实现set方法注入。

    假定有以下这些集合数据，并且对这些集合数据生成set方法。

    <bean id="accountService2" class="com.xiong.service.impl.AccountServiceImpl2">
    <property name="strs">
        <array>
            <value>AAA</value>
            <value>BBB</value>
            <value>CCC</value>
        </array>
    </property>
    <property name="lists">
        <list>
            <value>AAA</value>
            <value>BBB</value>
            <value>CCC</value>
        </list>
    </property>
    <property name="sets">
        <set>
            <value>AAA</value>
            <value>BBB</value>
            <value>CCC</value>
        </set>
    </property>
    <property name="maps">
        <props>
            <prop key="testProp"></prop>
            <prop key="testProp2"></prop>
        </props>

    </property>
    <property name="props">
        <map>
            <entry key="key A" value="val AA"></entry>
            <entry key="key B">
                <value>bbb</value>
            </entry>
        </map>
    </property>
</bean>

    用于给List结构集合注入的标签：（List结构集合包括List，Array，Set）

    • list
    • array
    • set

    用于给Map结构集合注入的标签：（Map结构集合包括Map，Properties）

    • map
    • props

    总结：结构相同，标签可以互换。List结构或者Map结构的数据下面的那几个标签全部都可以使用。

    例如上面map和props的注入，就是互换了顺序，但是依然可以正常执行。
