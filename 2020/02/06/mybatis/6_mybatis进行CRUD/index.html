    mybatis的CRUD操作

    1. C 创建记录

    StudentDao接口中添加方法：

    void saveStudent(Student student);

    xml文件中增加一个SQL查询：

    <!--    保存用户-->
<insert id="saveStudent" parameterType="com.xiong.domain.Student">
    <!-- 配置插入操作后，获取插入数据的ID -->
    <!--    keyProperty对应的属性名称 
           order 表示什么时候进行这个操作 
           resultType 表示返回值类型-->
    <selectKey keyProperty="id" order="AFTER" resultType="int" keyColumn="id">
        select last_insert_id();
    </selectKey>
    insert into student(name, age, sex, address, math, english) values(#{name}, #{age}, #{sex}, #{address}, #{math}, #{english});
</insert>

    测试代码如下：

    /*
 * 测试保存用户
 * */
@Test
public void testSave() {
    Student student = new Student();
    student.setName("mybatis");
    student.setAge(24);
    student.setSex("男");
    student.setAddress("呵呵");
    student.setMath(100);
    student.setEnglish(100);
    // 添加了selectKey之后，会获取得到这个id的，在执行savestudent操作前后分别打印出两个结果，可以发现一开始是null，后来是12，也就是被写进去了
    System.out.println(student.getId());
    // 执行保存方法
    studentDao.saveStudent(student);
    System.out.println(student.getId());
}

    2. R 读取记录

    2.1 根据id查询

    这种调用方法很朴素，就是向其中传一个参数，

    Student findByID(Integer id);

    xml文件定义为：

    <select id="findByID" parameterType="int" resultType="com.xiong.domain.Student">
    <!-- 注意，这里的id只是用来表示参数的，因为参数只有一个，所以随便写什么都可以 -->
    select * from student where id=#{id};
</select>

    调用的java方法为：

    @Test
public void testFind() {
    Student student = studentDao.findByID(10);
    System.out.println(student);
}

    2.2 根据用户名模糊查询

    StudentDao接口中定义一个接口：

    List<Student> findByName(String username);

    xml文件中定义的SQL查询为：

    <!--    根据名称进行模糊查询-->
<select id="findByName" parameterType="String" resultType="com.xiong.domain.Student">
    select * from student where name like #{name};
    <!--        select * from student where name like '%${value}%';-->
</select>

    测试的java类如下：

        @Test
    public void testFindByName() {
         List<Student> students = studentDao.findByName("%马%");
        // 下面的这种写法在xml的SQL语句中就已经加了百分号，所以这里就不用再加百分号占位符了
//        List<Student> students = studentDao.findByName("马");
        for (Student student : students) {
            System.out.println(student);
        }
    }

    2.3 使用聚合函数

    聚合函数实际上就是一个统计的函数,，就相当于是一个普通的查询，在调用的时候可以直接拿到结果。

    在StudentDao接口中定义下面的一个接口：

    int findTotal();

    在xml文件中实现SQL查询：

    <!--    获取用户的总记录行数-->
<select id="findTotal" resultType="int">
    select count(id) from student;
</select>

    在测试java文件中调用：

    @Test
public void testTotal() {
    System.out.println(studentDao.findTotal());
}

    2.4 使用queryVo条件查询

    首先定义QueryVo类，因为条件查询也是需要一个Student类的对象作为查询条件，因此这里的queryVo相当于只是一个包装对象，把Student的对象包装在queryVo对象中，查询的时候从包装中拆解出来并使用。

    根据queryVo中的查询条件来查询用户，还是通过名称来查询。只不过查询的对象被封装到一个对象：QueryVo中

    List<Student> findStudentByVo(QueryVo vo);

    xml中的使用方式为：

    <!--    根据QueryVo的条件查询用户-->
<select id="findStudentByVo" 
        parameterType="com.xiong.domain.QueryVo" 
        resultType="com.xiong.domain.Student">
    select * from student where name like #{student.name};
</select>

    测试java函数：

    @Test
public void testFindByVo() {
    QueryVo vo = new QueryVo();
    Student student = new Student();
    student.setName("%马%");
    vo.setStudent(student);
    List<Student> students = studentDao.findStudentByVo(vo);
    for (Student student1 : students) {
        System.out.println(student1);
    }
}

    3. U 更新记录

    在StudentDao接口中定义一个接口：

    void updateStudent(Student student);

    下xml文件中对接口进行SQL实现：

    要传进来的参数就是一个Student类型的对象，根据传进来对象的id来进行更新，即 where id = #{id}

    <update id="updateStudent" parameterType="com.xiong.domain.Student">
    update student set name=#{name}, age=#{age}, sex=#{sex}, address=#{address}, math=#{math}, english=#{english} where id=#{id};
</update>

    调用的java方法：

    @Test
public void testUpdate() {
    Student student = new Student();
    student.setId(10);
    student.setName("Update");
    student.setAge(24);
    student.setSex("男");
    student.setAddress("北京");
    student.setMath(23);
    student.setEnglish(44);

    studentDao.updateStudent(student);
}

    OGNI表达式

    Object Graphic Navigation Language

    对象图导航语言，通过对象的取值方法来获取数据，在写法上，把get给省略了。

    例如：获取用户名称：

    • 类中的写法：user.getUsername()
    • OGNL表达式写法：user.username

    为什么在mybatis中可以直接写username呢？是因为在parameterType中已经提供了属性所属的类，因此此时不需要写对象名。

    mybatis的参数：

    1. parameterType（输入类型）

    2. 传递简单类型

    3. 传递pojo对象

      mybatis使用OGNL表达式解析对象字段的值，#{}或者${}括号中的值为pojo属性名称

    4. 传递pojo包装对象

      开发中通过pojo传递查询条件，查询条件是综合的查询条件，不仅包含用户查询条件，还包含了其他的查询条件（比如将用户购买的商品信息也作为查询条件，这时可以通过包装对象传入参数，Pojo类中包含pojo）需求是：根据用户名查询用户信息，查询条件放到QueryVo的user属性中。也就是将实体类对象包装起来做查询。

        输出简单类型必须查询出来的结果集有一条记录，最终将第一个字段的值转换为输出类型。使用session的selectOne可以查询单条记录。

      2. 输出pojo对象

      3. 输出pojo列表

    如果实体类与数据库的列名不一致，有两种解决思路：

    1. 就在SQL语句里面设置别名，相当于查询的结果与实体类的属性保持一致了，这样就可以令实体类的列名与数据库不一致。
    select id as userId, name as userName,....

    1. 另一种解决方式，就是配置查询结果的列名和实体类的属性名之间的对应关系。property中的是java实体类的列名，要严格区分大小写，后面的column表示的是数据库的列名。同时，resultType需要更换为resultMap，如下所示：

      使用第二种方式处理由于需要多余解析一个xml，因此执行效率不如第一种方式（直接在SQL语句上面起别名）。不过开发的效率会比较高，因为只需要改变一次，其他地方使用的时候只需要引用就可以了。
