    Spring中IOC的常用注解

    前一节中对于AccountService的XML配置为：

    <bean id="accountService" class="com.xiong.service.impl.AccountServiceImpl"></bean>

    常用注解

    1.1 用于创建对象

    1.1.1 Component

    作用与XML配置文件中编写一个<bean>标签的作用相同，在类名前面加上关键字@component

    作用：

    • 把当前类对象存入容器中

    属性：

    • value：用于指定bean的id，当不写的时候，默认值是当前类名，并且首字母小写

      例如，对于默认值，应该使用下面的方法调用：

      public static void main(String[] args) {
    // 1. 获取核心容器对象
    ApplicationContext ac = new ClassPathXmlApplicationContext("bean.xml");
    // 2. 根据id获取bean对象
    // 拿到Object类型并进行强制类型转换
    AccountService as = (AccountService) ac.getBean("accountServiceImpl");
    System.out.println(as);
}

      但是，如果要指定value的话，可以把@Component修改为：

      @Component(value = "accountService")

      然后再使用accountService获取bean对象。

    用法：

    在xml中指定，告知spring在创建容器的时候需要扫描的包，配置所需要的的标签不是在beans的约束中，而是一个名称为context名称空间和约束中，并且指定扫描的包。也就是在bean.xml文件中将原来的beans约束替换成下面这样：

    <beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:context="http://www.springframework.org/schema/context"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/context
        http://www.springframework.org/schema/context/spring-context.xsd">

    <!--加上这个之后就会去扫描整个com.xiong包下带有注解的类-->
    <context:component-scan base-package="com.xiong"></context:component-scan>

</beans>

    1.1.2 Controller

    一般用在表现层

    1.1.3 Service

    一般用在业务层

    1.1.4 Repository

    一般用在持久层

    第2、3、4个注解，他们的作用和属性与第一个是相同的，他们三个是spring提供的明确三层使用的注解，使得三层对象更加清晰。如果有些对象不属于三层中的任何一层，则使用Component来创建

    1.2 用于注入数据

    作用与在XML配置文件中的bean标签中写一个property标签的作用一样。用于注入数据的注解有以下这些：

    @Autowired @Qualifier @Resource @Value

    注意！！！

    • 前三个注解都只能注入其他bean类型的数据，
    • 基本类型和String类型的注入需要通过第四个注解来实现
    • 集合类型的注入只能通过XML来实现。

    下面详细介绍这些注解：

    1.2.1 Autowired

    作用：

    • 自动按照类型注入，只要容器中有唯一的一个bean对象类型和要注入的变量类型相匹配，就可以注入成功

    出现位置：

    • 可以是变量上，也可以是方法上。

      例如在这个AccountDaoImpl上面加上@Repository(value="accountDao")，指定其对象，然后再在service的实现中调用，即可正常获得对象。

      @Component(value = "accountService")
public class AccountServiceImpl implements AccountService {
    @Autowired
    private AccountDao accountDao;
    @Override
    public void saveAccount() {
        accountDao.saveAccount();
    }
}

    注意：

    • case1：如果IOC容器中没有任何bean的类型与要注入的变量类型匹配，则会报错

    • case2：如果IOC容器中有多个类型匹配时，先按照类型，圈定出来匹配的对象，然后使用变量名称作为bean的id，在圈定出来的对象中继续查找，如果有一样的，也可以注入成功，如果两个都不一样则会报错。

      例子：

    细节：

    • 使用注解注入时，set方法不是必须的。

    1.2.2 Qualifier

    作用：

    • 在按照类中注入的基础上再按照名称注入，它在给类成员注入时不能够单独使用，而必须和Autowired一起使用。但是在给方法参数注入时可以单独使用。

      例如在上面的那个例子中，可以在这里指定value=accountDao2，而accountDao2就是在前面

    属性：

    • value：用于指定注入bean的id

    1.2.3 Resource

    作用：

    • 直接按照bean的id进行注入，可以独立使用。使用一个Resource注解就相当于同时使用了Autowired和Qualifier两个注解

    属性：

    • name：用于指定bean的id

    注意：

    • 需要在pom文件中加上java.annotation包依赖

      <dependency>
    <groupId>javax.annotation</groupId>
    <artifactId>javax.annotation-api</artifactId>
    <version>1.3.1</version>
</dependency>

    1.2.4 Value

    作用：

    • 用于注入基本类型和String类型的数据

    属性：

    • value：用于指定数据的值。他可以使用spring中的SpEL（spring中的EL表达式）

      SpEL的写法：${表达式}

    1.3 用于改变作用范围

    作用与在bean标签中使用scope属性实现的功能一样

    1.3.1 Scope

    作用：

    • 指定bean的作用范围

    属性：

    • value：指定范围的取值，常用的取值有：
      1. singleton：单例
      2. prototype：多例

    1.4 与生命周期相关

    作用与在bean标签中使用init-method和destroy-method的作用一样

    1.4.1 PreDestroy

    作用：

    • 用于指定销毁方法

    1.4.2 PostConstruct

    作用：

    • 用于指定初始化方法

    使用下面的方式调用：

    public static void main(String[] args) {
    // 1. 获取核心容器对象
    ClassPathXmlApplicationContext ac = new ClassPathXmlApplicationContext("bean.xml");
    // 2. 根据id获取bean对象
    // 拿到Object类型并进行强制类型转换
    AccountService as = (AccountService) ac.getBean("accountService");
    System.out.println(as);
    as.saveAccount();
    ac.close();
}

    2. 配置bean.xml

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/context
        http://www.springframework.org/schema/context/spring-context.xsd">
</beans>

    然后在该标签里面配置Service。

    • 由于在accountService的实现中用到了accountDao，因此需要注入accountDao
    • 由于accountDao的实现中用到了QueryRunner，因此需要注入QueryRunner
    • 由于QueryRunner需要用到数据源，因此需要注入数据源，这里是通过构造函数注入的，并且是多例对象。如果是单例对象，并且多个线程同时使用，则可能会互相干扰。
    • 配置数据源的时候需要四大必备信息：
      1. 配置MySQL的驱动
      2. jdbc的URL：注意配置URL的时候需要把时区和编码设置一下。
      3. 用户名
      4. 密码

    3. 创建Account实体类

    由于数据库中account表中包含了id，name，money三个属性，因此在Account实体类中也添加这3个属性，并且实现相应的getter setter以及toString方法。

    public class Account implements Serializable {

    private Integer id;
    private String name;
    private Float money;
    //...相应的方法
}

    4. 创建AccountService业务层

    其中包含查询所有、查询单个、保存、更新、删除五种方法，接口如下：

    public interface AccountService {
    List<Account> findAllAccount();
    Account findAccountById(Integer accountId);
    void saveAccount(Account account);
    void updateAccount(Account account);
    void deleteAccount(Integer acccountId);
}

    具体的实现如下：其中所使用到的AccountDao将在下面进行定义

    public class AccountServiceImpl implements AccountService {

    private AccountDao accountDao;

    // 下面这个set方法让spring注入该对象
    public void setAccountDao(AccountDao accountDao) {
        this.accountDao = accountDao;
    }

    @Override
    public List<Account> findAllAccount() {
        return accountDao.findAllAccount();
    }

    @Override
    public Account findAccountById(Integer accountId) {
        return accountDao.findAccountById(accountId);
    }

    @Override
    public void saveAccount(Account account) {
        accountDao.saveAccount(account);
    }

    @Override
    public void updateAccount(Account account) {
        accountDao.updateAccount(account);
    }

    @Override
    public void deleteAccount(Integer acccountId) {
        accountDao.deleteAccount(acccountId);
    }
}

    5. 创建AccountDao持久层

    本次使用的案例是进行数据库的CRUD，因此直接通过业务层调用持久层的相关内容，因此也需要以下5个方法：

    public interface AccountDao {
    List<Account> findAllAccount();
    Account findAccountById(Integer accountId);
    void saveAccount(Account account);
    void updateAccount(Account account);
    void deleteAccount(Integer acccountId);
}

    其具体实现如下，需要通过QueryRunner的对象调用相关的方法。

    • query方法：可以用于查询的SQL语句执行，注意第二个参数是返回值类型，后面才是传入的参数。

      并且这里的返回值类型指定方式有以下两种：

      1. 单条数据：new BeanHandler<Account>(Account.class)
      2. 多条数据：new BeanListHandler<Account>(Account.class)

    • update方法：可以用于插入、更新、删除的SQL语句执行，注意参数在SQL语句中用问号？表示占位符，在后面可以用参数来补充。

    public class AccountDaoImpl implements AccountDao {

    private QueryRunner runner;

    public void setRunner(QueryRunner runner) {
        this.runner = runner;
    }

    @Override
    public List<Account> findAllAccount() {
        try{
            return runner.query("select * from account",
                                new BeanListHandler<Account>(Account.class));
        }catch (Exception e) {
            throw new RuntimeException(e);
        }
    }

    @Override
    public Account findAccountById(Integer accountId) {
        try{
            return runner.query("select * from account where id = ? ",
                                new BeanHandler<Account>(Account.class),
                                accountId);
        }catch (Exception e) {
            throw new RuntimeException(e);
        }
    }

    @Override
    public void saveAccount(Account account) {
        try{
            runner.update("insert into account(name,money)values(?,?)",
                          account.getName(),
                          account.getMoney());
        }catch (Exception e) {
            throw new RuntimeException(e);
        }
    }

    @Override
    public void updateAccount(Account account) {
        try{
            runner.update("update account set name=?,money=? where id=?",
                          account.getName(),
                          account.getMoney(),
                          account.getId());
        }catch (Exception e) {
            throw new RuntimeException(e);
        }
    }

    @Override
    public void deleteAccount(Integer accountId) {
        try{
            runner.update("delete from account where id=?",accountId);
        }catch (Exception e) {
            throw new RuntimeException(e);
        }
    }
}

    7. 换为注解配置

    在bean.xml文件中将Service和Dao对象的配置删掉，只使用一条配置语句来代替，就是告知spring在创建容器的时候要扫描的包。

    然后在持久层和业务层的实现中添加相应的注解，并且把实现中的set方法换成@Autowired注解就可以了。

    新的注解

    不管是基于XML还是基于注解的spring都离不开要在bean.xml文件中进行配置，但是如果想要把这个xml依赖给摆脱掉。

    首先需要创建一个配置类，作用与bean.xml是一样的。

    1. Configuration

    作用：

    • 指定当前类是一个配置类

    获取容器的时候需要通过AnnotationConfigApplicationContext来进行。

    例如上面那个，SpringConfiguration类，在创建核心容器的时候需要通过该方法来创建

    ApplicationContext ac = new AnnotationConfigApplicationContext(SpringConfiguration.class);

    细节：

    • 当配置类作为AnnotationConfigApplicationContext对象创建的对象时，可以不写这个注解。

    2. ComponentScan

    作用：

    • 用于通过注解指定spring在创建容器时要扫描的包

    属性：

    • value：与basePackages的作用一样，都是用于指定创建容器时要扫描的包。使用此注解就等同于在xml中配置。

      这个注解的作用就和下面xml文件中圈住的标签作用是一样的。

    3. Bean

    作用：

    • 作用就是用于把当前方法的返回值作为Bean对象，并且存入Spring的IoC容器中。

    属性：

    • name：用于指定Bean的id。例如下面创建数据源对象

      @Bean(name="dataSource")
public DataSource createDataSource() {
    try {
        ComboPooledDataSource ds = new ComboPooledDataSource();
        ds.setDriverClass("");
        ds.setJdbcUrl("");
        ds.setUser("");
        ds.setPassword("");
        return ds;
    } catch(Exception e) {
        throw new RuntimeException(e);
    }
}

    细节：

    • 使用注解配置方法时，如果方法有参数，spring会去容器中查找有没有可用的bean对象
    • 查找的方式和Autowired注解的作用是一样的，也就是自动按照类型注入。

    4. Import

    作用：

    • 用于导入其他的配置类，可以把其他的配置类加载到主配置类里面。

    属性：

    • value：在括号中指定字节码。如果有多个，可以将多个都放进大括号里面。

      也就相当于把分配置文件加到主配置文件里面。

    • 使用Import注解之后，有Import的类就叫做父配置类，导入的其他类叫做子配置类。

    5. PropertySource[s]

    作用：

    • 就是用于指定properties文件的位置

    属性：

    • value：指定文件名称和文件路径，其中的classpath关键字表示类路径下。
