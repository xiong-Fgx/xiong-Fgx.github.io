    项目前置配置

    1. 业务层及其代码

    在业务层接口中创建三个接口，分别代表三种不同的业务层代码

    • 无返回值、无参数：void saveAccount()
    • 无返回值、有参数：void updateAccount(int i)
    • 有返回值、无参数：int deleteAccount()
    public interface AccountService {

    void saveAccount();

    void updateAccount(int i);

    int deleteAccount();
}

    这些类的实现方法写在对应的impl文件中，也就是简单地打印一句话。

    public class AccountServiceImpl implements AccountService {
    @Override
    public void saveAccount() {
        System.out.println("执行了保存");
    }
    @Override
    public void updateAccount(int i) {
        System.out.println("执行了更新" + i);
    }
    @Override
    public int deleteAccount() {
        System.out.println("执行了删除");
        return 0;
    }
}

    2. 具有公共代码的类

    这里设置一个打印日志的公共代码，主要用于在执行业务层方法之前或者之后能够打印一些信息：

    public class Logger {
    /*
    * 用于打印日志，计划让其在切入点方法执行之前执行（切入点方法就是业务层方法）
    * */
    public void printLog() {
        System.out.println("Logger类中的printLog方法开始记录日志");
    }
}

    基于xml的AOP配置

    1. 配置pom.xml文件

    需要添加以下两个依赖，第一个是spring IOC的依赖，第二个是AOP的依赖

    <dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-context</artifactId>
    <version>5.2.3.RELEASE</version>
</dependency>

<dependency>
    <groupId>org.aspectj</groupId>
    <artifactId>aspectjweaver</artifactId>
    <version>1.8.7</version>
</dependency>

    2. 配置bean.xml文件

    导入AOP的约束

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:aop="http://www.springframework.org/schema/aop"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/aop
        http://www.springframework.org/schema/aop/spring-aop.xsd">
</beans>

    需要先配置spring的IOC，把service对象配置进来

    <bean id="accountService" class="com.xiong.service.impl.AccountServiceImpl"></bean>

    配置AOP的相关信息：

    1. 配置通知Bean交给Spring管理，即配置Logger类

      <bean id="logger" class="com.xiong.utils.Logger"></bean>

    2. 使用aop:config标签表明AOP的配置开始

    3. 使用aop:aspect标签表明开始配置切面

      • id属性：给切面提供一个唯一标识
      • ref属性：指定通知类的bean id

    4. 在aop:aspect标签的内部使用对应的标签来配置通知的类型，并且建立通知方法和切入点方法的关联

      现在的示例是让pringLog方法在切入点方法执行之前执行，所以是前置通知，使用aop:before，表示配置前置通知。

      • method属性：用于指定logger类中哪个方法是前置通知

      • pointcut属性：用于指定切入点表达式，该表达式的含义指的是对业务层中哪些方法增强

        关键字：execution(表达式)。

        表达式：访问修饰符 返回值 包名.包名....类名.方法名(参数列表)

        下面是一个标准的表达式写法，比如要对saveAccount进行增强

        public void com.xiong.service.impl.AccountServiceImpl.saveAccount()

    第234条放一起配置出来的结果如下：

    <aop:config>
    <aop:aspect id="logAdvice" ref="logger">
        <aop:before method="printLog" 
                    pointcunt="execution(public void com.xiong.service.impl.AccountServiceImpl.saveAccount())"></aop:before>
    </aop:aspect>
</aop:config>

    通过下面的方法进行测试：

    public class AOPtest {
    public static void main(String[] args) {
        // 1. 获取容器
        ApplicationContext ac = new ClassPathXmlApplicationContext("bean.xml");
        // 2. 获取对象
        AccountService as = (AccountService) ac.getBean("accountService");
        // 3. 执行方法
        as.saveAccount();
        as.updateAccount(1);
        as.deleteAccount();
    }
}

    3. 切入点表达式写法

    1. 标准的表达式写法：

      访问修饰符 返回值 包名.包名....类名.方法名(参数列表)

      例如：

      public void com.xiong.service.impl.AccountServiceImpl.saveAccount()

    2. 通配符的使用：

      • 访问修饰符可以省略，可以将上面的标准写法简化为：

        void com.xiong.service.impl.AccountServiceImpl.saveAccount()

      • 返回值可以使用通配符 * 表示任意返回值，可以将上面的标准写法简化为：

        * com.xiong.service.impl.AccountServiceImpl.saveAccount()

      • 包名可以使用通配符 *. 表示任意包，但是有几级包就需要些几个 *.，因此可以简化为：

        * *.*.*.*.AccountServiceImpl.saveAccount()

      • 包名可以使用 .. 表示当前包及其子包，因此可以简化为：

        * *..AccountServiceImpl.saveAccount()

      • 类名和方法名都可以实现通配

        * *..*.saveAccount() // 通配类名
* *..*.*() // 通配类名和方法名

        如果同时通配成上面的形式，则只能匹配两个方法，因为这里的参数是空的。

      • 参数列表可以写数据类型，也可以通配

        1. 基本类型直接写名称
        2. 引用类型写 包名.类名 的方式
        3. 使用 .. 进行通配
        * *..*.*(int)
* *..*.*(java.lang.String)
* *..*.*(..)

    实际开发的时候，切入点表达式的通常写法是：

    • 切到业务层实现类实现下的所有方法，对于这里就是：

      * com.xiong.service.impl.*.*(..)

    4. 配置四种通知类型

    要配置不同类型的通知，首先需要在公共代码中写四个方法：一个小case是，异常通知和后置通知之中只能执行一个。

    public class Logger {
    // 前置通知
    public void beforePrintLog() {
        System.out.println("这是一条前置通知");
    }
    // 后置通知
    public void afterReturningPrintLog() {
        System.out.println("这是一条后置通知");
    }
    // 异常通知
    public void afterThrowingPrintLog() {
        System.out.println("这是一条异常通知");
    }
    // 最终通知
    public void afterPrintLog() {
        System.out.println("这是一条最终通知");
    }
}

    剩下的事情就是进行xml文件的配置，其实也就是把aop的标签改成对应的标签就可以了，然后指定method

    4.1 前置通知

    <aop:before method="beforePrintLog" 
            pointcut="execution(* com.xiong.service.impl.*.*(..))">
</aop:before>

    4.2 后置通知

    <aop:after-returning method="afterReturningPrintLog" 
                     pointcut="execution(* com.xiong.service.impl.*.*(..))">
</aop:after-returning>

    4.3 异常通知

    <aop:after-throwing method="afterThrowingPrintLog" 
                    pointcut="execution(* com.xiong.service.impl.*.*(..))">
</aop:after-throwing>

    4.4 最终通知

    <aop:after method="afterPrintLog" 
           pointcut="execution(* com.xiong.service.impl.*.*(..))">
</aop:after>

    5. 配置环绕通知

    在Logger类中添加一个环绕通知的方法：

    public void aroundPringLog() {
    System.out.println("这是一条环绕通知");
}

    然后在bean.xml文件中进行配置，环绕通知使用的标签是aop:around

    <aop:around method="aroundPringLog" pointcut-ref="pt1"></aop:around>

    可以重新运行一下，发现结果如下：

    也就是说，当配置了环绕通知之后，切入点方法没有执行，而通知方法执行了。

    分析：对比动态代理部分的代码，发现这是因为环绕通知其实就是整个增强后的方法，包括增强部分和原始部分，其中，在整个环绕通知内部，有显式调用原始业务层部分的代码。在这里没有，因此只执行了环绕通知的部分。

    解决：spring框架提供了一个接口：proceedingJoinPoint，该接口有一个方法proceed，此方法就相当于明确调用切入点方法。该接口可以作为环绕通知的方法参数，在程序执行时，spring框架会提供该接口的实现类以供使用，具体实现类似于之前写的动态代理。

    public Object aroundPringLog(ProceedingJoinPoint pjp) {
    Object returnValue = null;
    try {
        // 得到方法执行所需的参数
        Object[] args = pjp.getArgs();
        System.out.println("这是一条环绕通知中的前置通知");
        // 显式调用业务层方法（切入点方法）
        returnValue = pjp.proceed(args);
        System.out.println("这是一条环绕通知中的后置通知");
        return returnValue;
    } catch (Throwable t) {
        System.out.println("这是一条环绕通知中的异常通知");
        throw new RuntimeException(t);
    } finally {
        System.out.println("这是一条环绕通知中的最终通知");
    }
}

    其实，环绕通知是spring框架提供的一种可以手动在代码中控制增强方法何时执行的方式。
