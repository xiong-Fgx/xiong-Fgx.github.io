    注解开发

    1. 环境搭建

    首先创建一个mysql的配置文件，之前从mysql中读取配置都是通过直接在里面写死实现的，后面开发的过程中可以将这些配置信息写到一个properties文件中，以便于修改后不需要重新编译。

    jdbc.driver=com.mysql.cj.jdbc.Driver
jdbc.url=jdbc:mysql://localhost:3306/db1?useUnicode=true&useJDBCCompliantTimezoneShift=true&useLegacyDatetimeCode=false&serverTimezone=UTC
jdbc.username=root
jdbc.password=

    要注意的是，之前写在xml文件中的符号 & 由于识别不出来，所以要写成 &amp; 但是在这里写到properties文件中的话是可以识别出来的，因此要把之前的 &amp; 全部换成 & 。

    然后，主配置文件中，将从这个properties文件中读取相应的参数完成mysql的配置，其中的各项都比较清晰，引入一个外部properties配置文件，并且从这个配置文件中读取相应的参数，然后就是设置别名的部分，就是从指定的包位置找到别名设置，以后再用的时候就不需要写全称了。

    <?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <!--引入外部配置文件-->
    <properties resource="jdbcConfig.properties"/>

    <!--配置别名-->
    <typeAliases>
        <package name="com.xiong.domain"/>
    </typeAliases>

    <!--配置环境-->
    <environments default="mysql">
        <environment id="mysql">
            <transactionManager type="JDBC"/>
            <dataSource type="POOLED">
                <property name="driver" value="${jdbc.driver}"/>
                <property name="url" value="${jdbc.url}"/>
                <property name="username" value="${jdbc.username}"/>
                <property name="password" value="${jdbc.password}"/>

            </dataSource>
        </environment>
    </environments>
    <!-- 引入指定带有注解的dao接口所在位置 -->
    <mappers>
        <!--表示dao接口的位置-->
        <package name="com.xiong.dao"/>
    </mappers>
</configuration>

    2. 单表CRUD（代理Dao）

    在mybatis中，CRUD一共有4个注解，分别为：

    • @Select
    • @Insert
    • @Update
    • @Delete

    只需要在接口文件中将注解写在接口上，然后把SQL语句作为注解的参数，就可以了

    测试的java方法为：

    public static void main(String[] args) throws IOException {
    // 1. 获取字节输入流
    InputStream in = Resources.getResourceAsStream("SqlMapConfig.xml");
    // 2. 根据字节输入流构建SqlSessionFactory
    SqlSessionFactory factory = new SqlSessionFactoryBuilder().build(in);
    // 3. 根据SqlSessionFactory生产一个SqlSession
    SqlSession session = factory.openSession();
    // 4. 使用SqlSession获取Dao的代理对象
    StudentDao studentDao = session.getMapper(StudentDao.class);
    // 5. 执行Dao的方法
    List<Student> students = studentDao.findAll();
    for (Student student : students) {
        System.out.println(student);
    }
    // 6. 释放资源
    session.close();
    in.close();
}

    至于其他的操作，在StudentDao接口中，注解写成下面这样：

    @Select("select * from student")
List<Student> findAll();

@Insert("insert into student(name, address) values(#{name}, #{address})")
void saveUser(Student student);

@Update("update student set name=#{name}, address=#{address} where id=#{id}")
void update(Student student);

@Delete("delete from student where id=#{id}")
void delete(Integer id);

// 根据ID查询用户
@Select("select * from student where id=#{userId}")
Student findById(Integer userId);

// @Select("select * from student where name like #{name}")
@Select("select * from student where name like '%${value}%'")
List<Student> findStudentByName(String name);

@Select("select count(*) from student")
int findStudentCount();

    3. 多表查询操作

    3.1 一对一

    准备一个新的数据类Account，在其中添加account表中的三个属性，以及其对应的Student对象。为其设置getter和setter方法

    这时，需要在查询account表的时候顺带把student信息也查询出来，则注解应该这样写：

    @Select("select * from account")
@Results(id="accountMap", value = {
    @Result(id=true, column = "id", property = "id"),
    @Result(column = "uid", property = "uid"),
    @Result(column = "money", property = "money"),
    // 表示通过uid查找student表，指向com.xiong.dao.StudentDao.findById
    @Result(column = "uid", 
            property = "student", 
            one=@One(
                select = "com.xiong.dao.StudentDao.findById", 
                fetchType= FetchType.EAGER))

})
List<Account> findAll();

    注意，在表示student对象的结果的时候，需要使用一个one的参数，该参数使用注解 @One，并且此注解中也有两个参数，即其调用的方法注解以及加载机制（立即加载或者延迟加载）。

    3.2 一对多

    与一对一类似，这里的一对多是通过在Student类中增加一个List的Account对象来实现的。

    同样，将相关的方法注解写成下面这样：

    @Select("select * from student")
@Results(id="userMap", value = {
    @Result(id=true, column = "id", property = "id"),
    @Result(column = "name", property = "name"),
    @Result(column = "age", property = "age"),
    @Result(column = "sex", property = "sex"),
    @Result(column = "address", property = "address"),
    @Result(column = "math", property = "math"),
    @Result(column = "english", property = "english"),
    @Result(column = "id", 
            property = "accounts", 
            many = @Many(
                select = "com.xiong.dao.AccountDao.findAccountByUid", 
                fetchType = FetchType.LAZY))
})
List<Student> findAll();

    注意，其中的many就表示的是一对多的“多”关系，它通过column=”id”这个属性作为参数来执行操作的参数，@many 注解中的select表示执行方法的全限定类名，fetchType就是加载方式，即延迟加载或者立即加载。

    4. 缓存的配置

    一级缓存：自动开启的，不需要过多关注

    @Test
public void testFindAccount() {
    List<Account> accounts = accountDao.findAll();
    List<Account> accounts1 = accountDao.findAll();
    System.out.println(accounts==accounts1);
}

    返回结果为true。

    二级缓存

    在配置文件中配置二级缓存（不过这个二级缓存默认是开启的）

    <settings>
    <setting name="cacheEnabled" value="true"/>
</settings>

    并且在对应的接口前面添加一个注解，表示使用二级缓存

    @CacheNamespace(blocking = true)

    然后，测试下面的这个方法

    public static void main(String[] args) throws IOException {
    // 1. 获取字节输入流
    InputStream in = Resources.getResourceAsStream("SqlMapConfig.xml");
    // 2. 根据字节输入流构建SqlSessionFactory
    SqlSessionFactory factory = new SqlSessionFactoryBuilder().build(in);
    // 3. 根据SqlSessionFactory生产一个SqlSession
    SqlSession session = factory.openSession();
    // 4. 使用SqlSession获取Dao的代理对象
    StudentDao studentDao = session.getMapper(StudentDao.class);
    // 5. 执行Dao的方法
    Student students = studentDao.findById(3);
    session.close();
    System.out.println("--第一次查询执行完毕--");
    SqlSession session1 = factory.openSession();
    StudentDao studentDao1 = session1.getMapper(StudentDao.class);
    Student students1 = studentDao1.findById(3);
    System.out.println("--第二次查询执行完毕--");
    // 6. 释放资源
    session1.close();
    in.close();
}

    可以从返回结果中看到只执行了一次查询操作
