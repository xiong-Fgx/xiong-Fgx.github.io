    页面布局的三大核心：盒子模型、浮动、定位

    布局的过程：

    1. 先准备好相关的网页元素，其中元素基本上都是盒子Box

    盒子模型的组成部分

    盒子模型就是将HTML页面中的布局元素看作是一个矩阵的盒子，即装内容的盒子。

    CSS盒子模型封装周围的HTML元素，包括：边框、外边距、内边距、实际内容。

    • margin（外边距）
    • border（边框）
    • padding（内边距）
    • content（实际内容）

    1. border

    1.1 基础写法

    div {
    border-color: #00ffff;
    border-style: double;
    border-width: 5px;
}

    可以看出，边框可以设置的属性有三个，这三个属性可以进行简写，并且三个属性的顺序没有要求。

    div {
    border: 5px solid red;
}

    此外，还可以只设置单个边框的属性，比如上下左右这四个边框的属性可以分别设置。

    div {
    border-top: 1px solid red;
}

    1.2 三大基本属性

    • 粗细 border-width

      一般使用精确的像素值来设置

    • 样式 border-style

      有多种设置的值

      参数 类型
      none 无边框，此时指定border-width无效
      hidden 隐藏边框
      dotted 点线
      dashed 虚线
      solid 实线
      double 双线
      groove 3D凹槽
      ridge 菱形边框
      inset 3D凹边
      outset 3D凸边

    • 颜色 border-color

    1.3 设置边框合并

    如果是两个边框靠在一起，那么他们的边框将不会重叠显示，而是分别显示，因此相邻位置的边框宽度会变成原来的2倍。所以需要通过一个属性来设置边框相关的信息。其中的collapse表示合并的意思。

    div {
    border-collapse: collapse;
}

    1.4 边框大小不包含在盒子内

    默认情况下：

    设置了盒子大小为 200*200 ，此时在设置边框大小为10px，最终展示出来包含边框的盒子的宽高为 220*220。

    2. padding

    padding在设置的时候可以只设定1个数字，也可以指定两个数字。

    • 如果只设定一个数字，表示上下左右的距离都是这个值
    • 如果设置两个数字，则其中第一个数字表示上下的padding高度，第二个数字表示左右的padding宽度。
    div {
    padding: 0 20px;
}

    此外，如果设定了padding，那么盒子的大小也是会受到影响的。

    但是如果盒子本身没有指定宽高属性，则此时的padding不会影响盒子的大小。

    即使是设置宽度为100%，加上padding之后也是会影响大小的。

    3. margin

    该属性用于设置外边距，控制盒子与盒子之间的距离。

    关于margin可以设置上下左右四个位置的边距，因此在设置的时候需要注意给哪个盒子设置。

    • margin-top
    • margin-bottom
    • margin-left
    • margin-right

    如果使用2个数值来设定，则为上下、左右边距

    如果使用4个数值来设定，则4个值的顺序为：上右下左（从上面开始顺时针转）

    3.1 块级盒子水平居中

    外边距可以让块级盒子水平居中，但是必须要满足两个条件

    1. 盒子必须指定宽度（width）
    2. 盒子左右的外边距设置为auto

    例如下面这个图中，将div设置为绿色，并且让其居中显示。

    div {
    background-color: green;
    width: 300px;
    margin: 0 auto;
}

    对于行内元素、行内块水平居中，需要对其父元素添加 text-align: center 属性。

    3.2 外边距合并

    外边距合并有下面的两种情况：

    • 相邻块元素垂直外边距的合并

      对于两个垂直放置的块元素，其上下边界会发生合并，只保留值较大的一个。比如设置下面的CSS样式：

      div {
    background-color: blue;
    height: 100px;
}
.shang {
    margin-bottom: 100px;
}
.xia {
    margin-top: 200px;
}

      在HTML页面中设置

      <div class="shang"></div>
<div class="xia"></div>

      会得到：

      两个div之间的距离是200px，也就是只取了较大值。

      解决方法：在相邻的情况下尽量只给一个盒子添加margin值

    • 嵌套块元素垂直外边距的塌陷

      对于两个有嵌套关系（父子元素）的块元素，父子元素都有上外边距，此时父元素会塌陷较大的外边距值。只要给子元素添加margin-top值，就一定会出现塌陷的现象。

      示意图如下：

      .father {
    width: 400px;
    height: 400px;
    background-color: purple;
    margin-top: 50px;
    /* padding: 1px; */
}
.son {
    width: 200px;
    height: 200px;
    background-color: pink;
    margin-top: 100px;
}
          <div class="father">
        <div class="son"></div>
    </div>

      这里上边距变成了100

      解决方法：

      1. 为父元素定义上边框
      2. 为父元素定义上内边距
      3. 为父元素添加：overflow:hidden

      加上之后显示就恢复正常了，此时外盒子的上边距为50，恢复正常

    3.3 清除内外边距

    对于网页元素来说，大多都是具有外边距的，因此可以通过设置通配符选择器的方式将默认内外边距给清除掉。

    * {
    padding: 0;
    margin: 0;
}

    对于行内元素而言，为了照顾兼容性，尽量只设置左右内外边距，不要设置上下内外边距。但是转换为块级和行内块元素就可以了。

    4. margin与padding

    一般而言，设置左右的边距使用padding，设置上面的边距使用margin-top。

    因为padding是算在盒子内部的，margin不算在盒子内部，需要占据额外的空间。

    圆角边框

    CSS3之后新增的3个样式：圆角边框、盒子阴影、文字阴影

    使用方式：设置 border-radius: length 属性即可。其中的length可以是精确的px也可以是百分比。

    通过这种方式可以设置圆角矩形，也可以设置圆形。

    该参数可以设置为1个值，2个值，4个值，分别表示左上、右上、右下、左下四个角（从左上角开始顺时针转动）

    盒子阴影

    CSS3版本之后出现的新特性，作用是给盒子加上阴影。通过 box-shadow 属性进行设置。

    注意：盒子阴影不会占用空间，不会影响其他盒子的排列

    该属性需要设置的参数为：

    • h-shadow：（必须）水平偏移距离
    • v-shadow：（必须）垂直偏移距离
    • blur：（选）模糊的距离
    • spread：（选）阴影的尺寸
    • color：（选）阴影颜色
    • inset：（选）阴影外置或者内置，默认情况下是外置的，但是不能写outset，否则会出错

    例如下面这样：

    div {
    width: 50px;
    height: 50px;
    background-color: pink;
    box-shadow: 10px 10px 10px 10px #C3C3C3;
}

    如果想要做成从中间往外发散的形式，则应该将前两个属性改为0

    div {
    width: 50px;
    height: 50px;
    background-color: pink;
    box-shadow: 0 0 10px 10px #C3C3C3;
}

    文字阴影

    文字阴影通过 text-shadow 属性进行设置。

    • h-shadow：水平偏移量
    • v-shadow：垂直偏移量
    • blur：模糊距离
    • color：颜色

    例如对下面的文字进行阴影设置

    div {
    text-shadow: 5px 5px 5px black;
}

    浮动

    1. 传统网页布局的三种方式（PC网页）

    网页布局的本质：使用CSS摆放盒子。网页布局方式有以下三种，实际开发的时候， 一个网页基本上都包含了这三种布局方式，移动端则会使用新的布局方式。

    1. 普通流（标准流/文档流）

      将标签按照默认规定好的方式排列，比如：

      • 块级元素独占一行，从上到下排列
      • 行内元素按照顺序从左到右排列，碰到父元素的边缘后会自动换行

    2. 浮动

    3. 定位

    通过标准流方式来实现某些效果是比较复杂的，比如：

    • 一行内显示多个块元素
    • 两个盒子实现左右对齐

    浮动可以改变元素默认的排列方式。

    网页排列第一准则：多个块级元素纵向排列使用标准流实现，多个块级元素横向排列使用浮动实现。

    2. 浮动的相关概念

    浮动：float属性用于创建浮动框，将其移动到一遍，直到左边缘或者右边缘触及包含块或另一个浮动框的边缘。

    float属性的取值有3个：

    • none：元素不浮动
    • left：元素向左浮动
    • right：元素向右浮动

    需要注意的是：浮动默认情况下是紧紧贴在一起的，中间没有间隙。

    div {
    float: 属性值;
}
    .zuo {
    float: left;
}

.you {
    float: right;
}

    设置完css之后，再定义两个相对应的盒子，即可将盒子左右放置。

    <div class="zuo">文字</div>
<div class="you">文字</div>

    3. 浮动的特点

    3.1 浮动元素会脱离标准流

    也就是说：浮动的盒子不再保留原先的位置。这样，这个盒子原来的位置就会空出来，其他的元素就会占据它原来的位置。

    举例说明，如果在同一个父亲下的两个盒子一个浮动一个未浮动，则浮动的盒子会压住未浮动的盒子。

    比如下面这个效果图，浮动的红色div会压住下面标准的粉色div

    div {
    width: 200px;
    height: 250px;
    background-color: pink;
    font-size: 20px;
    line-height: 50px;
}

.zuo {
    float: left;
    width: 100px;
    height: 100px;
    background-color: red;
}

    3.2 浮动的元素会在一行内显示并且元素顶部对齐

    如果多个盒子都设置了浮动，那么他们会显示在一行内，并且其顶端与顶部对齐，如果一行内装不下这些盒子，则会另起一行进行对齐。

    3.3 浮动的元素会具有行内块元素的特性

    对于一些行内元素而言，是不能设置宽高的，但是如果加了float属性之后，将其设置为浮动元素，此时宽高的设置就会生效，该元素就具有了行内块元素的特性。

    即：如果行内元素有了浮动，就不需要将其转换为行级、行内块元素就可以直接设置宽度和高度。

    同样，对于块级元素，设置了浮动之后也会具有行内块元素的特征。对于宽高的实际情况，需要经过行内块元素的实际情况决定。

    4. 浮动元素位置约束

    为了约束浮动元素的位置，经常采用的策略为：

    先用标准流的父元素排列上下位置，之后内部子元素采取浮动排列左右位置，符合网页布局第一准则。

    也就是用一个父盒子框柱，子元素在父盒子内部浮动。

    5. 注意事项

    1. 一般来说是将浮动和标准流的父盒子进行搭配使用

    2. 一个元素浮动，那么他的兄弟元素也应当浮动。（一浮全浮）

      注意，浮动的盒子只会影响浮动盒子后面的标准流，不会影响其前面的标准流，也就是说：

      • 标准流写在前面，浮动盒子写在后面：此时浮动不会压住标准流
      • 浮动盒子写在前面，标准流写在后面：此时浮动会压住标准流

    6. 清除浮动

    6.1 问题引入

    先看一个应用场景，由于在父元素盒子里面的内容不确定有多少，因此无法准确指定盒子大小。

    比如对于长度不确定的新闻，无法准确指定装新闻的盒子的大小。

    如果不指定父盒子的高度，并且将浮动的元素加入到父盒子之中，此时的父盒子高度会出现问题。比如下面：

    .father{
    width: 500px;
    border: 1px solid black;

}

.son1 {
    background-color: red;
    float: left;
    height: 50px;
    width: 100px;
}

.son2 {
    background-color: blue;
    float: left;
    height: 60px;
    width: 150px;
}

    总结一下，清除浮动的原因：

    1. 父级没有高度
    2. 子盒子浮动
    3. 影响下面的布局

    6.2 清除浮动本质

    清除浮动的本质就是清除浮动元素造成的影响。

    如果父盒子本身有高度，则不需要清除浮动。

    清除完浮动之后，父亲就会根据浮动的子盒子自动检测高度，此时由于父亲有了高度，下面的标准流就不会出现显示位置异常的问题了。

    清除浮动使用 clear 属性设置

    • left：清除左侧浮动的影响
    • right：清除右侧浮动的影响
    • both：同时清除两侧浮动的影响（开发中常用）

    清除浮动的策略：闭合浮动

    6.3 方法1：额外标签法

    额外标签法也称为隔墙法，做法是在浮动元素末尾加一个空标签，比如：

    <div style=" clear: both "></div>

    也可以加一个其他标签，比如：

    <br/>

    也就相当于是在浮动元素的底下再加一个标准的块级元素，此时这个标签会被放在浮动元素下面。

    优点：通俗易懂，书写方便

    缺点：会添加很多无意义的标签，结构化较差

    6.4 方法2：父元素overflow

    给父元素添加一个overflow属性，将其属性值设置为以下三种均可：

    • hidden（常用）
    • auto
    • scroll
    .father {
    overflow: hidden;
}

    优点：代码简洁

    缺点：无法显示溢出的部分

    6.5 方法3：after伪元素

    给父元素添加相关的属性：

    .clearfix:after { /* :after 是伪元素 */
    content: "";
    display: block; /* 转成块元素 */
    height: 0;
    clear: both;
    visibility: hidden;
}

/* IE6 7专有 */
.clearfix {
    *zoom: 1;
}

    6.6 方法4：双伪元素

    也是给父元素添加相关属性。只不过这里用了两个伪元素

    .clearfix:before, 
.clearfix:after {
    content: "";
    display: table;
}

.clearfix:after {
    clear: both;
}

.clearfix {
    *zoom: 1;
}
